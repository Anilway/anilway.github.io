从这一篇开始，我准备把目前学习过的JS方面的理论知识做一个总结，梳理知识框架。计划会用到思维导图、视频、音频、gif来呈现，希望简单直观。不过无论好坏，先做个坯子出来，再慢慢修改。

我们就从引入<script>开始。

1. 引入JS代码

通过< script >标签引入JS代码：

  • 嵌入式：直接在< script >标签内写代码，放在< head >标签或者< body >标签内
  • 外部式：< script src=”JS文件地址” >< /script >，放在< head >标签或者< body >标签内

Tips: 注意不要在外部式的script标签之间再嵌入js代码，会被忽略。

<script src="https://xxxx.com">
…code… //这部分代码会被忽略 
</script>


优先使用外部式引入：

在引入JS文件时有嵌入式和外部式，但更建议使用外部式。因为：

外部式优点 说明
更具维护性 可以不用管< html >中的标签，专注于JS代码本身
可缓存 浏览器在请求页面后，可将JS文件缓存。如果多个页面使用了同一JS文件，浏览器只需要请求一次
更具未来性 通过外部引用JS，可避免在XHTML中的严格写法< ![ CDATA[……] ] >；
对于不支持< script >标签的浏览器，可以不使用注释写法：< script >< !–代码– >< /script >；
（HTML和XHTML包含的外部文件的语法相同）


< script >的属性：

属性 说明
src 表示包含要执行代码的外部文件
type 可以看成是language属性的替代属性。表示编写代码使用的脚本语言的内容类型（或称为MIME类型）。
虽然text/javascript和text/ecmascript都已经不被推荐使用，但人们一直以来使用的还是text/javascript。实际上，服务器在传送JavaScript文件时使用的MIME类型通常是application/x-javascript，但在type中设置这个值却可能导致脚本被忽略。另外，在非IE浏览器中还可以使用以下值：application/javascript和application/ecmascript。考虑到约定俗成和最大限度的浏览器兼容性，目前type属性的值依旧还是text/javascript。不过这个属性不是必需的，如果没有指定这个属性，则其默认值仍为text/javascript。
language 已弃用。原来用于表示书写代码使用的脚本语言
defer 延迟加载脚本，只对外部脚本文件有效
aysnc 异步加载脚本，只对外部脚本文件有效
charset 表示通过src属性指定的代码字符集。大多数浏览器会忽略它的值，该属性很少用。


< noscript >：

用以在不支持< script >或者禁止加载< script >浏览器中显示替代文字。比如：

<noscript>
  <p>本页面需要浏览器支持（启用）JavaScript。</p>
</noscript>

2. JavaScript加载特点

特点 说明
一般浏览器会先加载JS文件，再加载页面内容 所以，最好将< script >标签放在< body >标签的最后，以减少页面加载时间，提升用户体验。
有多个JS文件时，浏览器会按顺序加载JS文件 所以，被依赖的JS文件放在前面，比如我引用jQuery.js，那它放前面；main.js放在后面

可以通过属性改变JS加载特性：

改变方式 说明
通过defer延迟加载 让浏览器先加载页面内容，后加载JS文件。由于只有部分浏览器支持该属性（chrome、IE4、FF3.5、safari），其他浏览器会忽略该属性。所以，最好把延迟脚本放在页面底部。
通过aysnc异步加载 浏览器可同时加载页面和JS文件。加载JS文件不会堵塞其他内容加载。要注意的是：浏览器不一定会按顺序加载JS文件，所以要保证JS文件之间没有关联性。

3.文档模式对JavaScript的影响

文档模式 说明
混杂模式 （又叫怪异模式） 如果没有声明文档类型，所有浏览器默认开启混杂模式。不同浏览器在这种模式下的差异很大，如果不使用某些hack技术，不同浏览器之间的行为根本没有一致性。混杂模式会让IE的行为与（包含非标准特性的）IE5相同。而标准模式则让IE的行为更为接近标准模式。
标准模式（又叫严格模式） 又叫严格模式。浏览器按W3C标准标准渲染页面。
准标准模式 又分为过度型和框架型。其中过度型既可以显示标准的，又可以显示不标准的。

tips：标准模式与准标准模式差不多，它们的差异几乎可以忽略不计。因此，有人提到标准模式时，有可能是这两种中的任何一种。

声明文档类型很重要：

不同的文档模式会影响CSS内容的呈现，在某些情况下也会影响到JS的解释执行。如果没有声明文档类型，所有浏览器会默认开启混杂模式，不同浏览器在混杂模式下差异非常大，跨浏览器没有一致性可言。而声明文档类型，就可以使浏览器开启标准模式，避免以上情况的出现。

  • 在所有HTML文档中规定< ！DOCTYPE >非常重要，这样浏览器就能了解预期的文档类型，告诉浏览器通过哪一种规范（文档类型定义DTD）解析文档。
  • ！DOCTYPE声明不属于HTML标签，它是一条指令，告诉浏览器编写页面所用的标记版本。
  • HTML4.01中的doctype要对DTD进行引用，因为HTML4.01基于SGML。
  • 而HTML5不基于SGML，因此不需要对DTD进行引用。但需要doctype来规范浏览器的行为，HTML5简化了这种声明。


4.总结

延伸阅读

名词解释：

  • DTD：文档类型定义。Document Type Definition
  • SGML：标准通用标记语言。Stanard Generalized Markup Language

<! DCOTYPE>为什么有个“！”？

<！DOCTYPE>声明不属于HTML标签，它是一条指令，告诉浏览器编写页面所用的标记版本。

<! DCOTYPE>的位置？

<！DOCTYPE>声明必须位于HTML文档的第一行，位于<html>标签之前

<! DCOTYPE>的作用？

告知浏览器用什么标准解析这个文档。DOCTYPE不存在或格式不正确会导致文档以兼容模式呈现。

HTML4.01与HTML5的DOCTYPE区别？

  • HTML5只有一种声明：<! DOCTYPE html>
  • HTML4.01有三种声明：strict（标准模式）、transitional（过渡型）、frameset（框架型）
  • HTML4.01基于SGML，需要对DTD进行引用，告知浏览器使用的文档类型。
  • HTML5不基于SGML，不需要对DTD进行引用，但需要用DOCTYPE规范浏览器行为。

SGML最为强大和古老，XML是SGML的一个子集，HTML最初也试图成为SGML的子集，但HTML4以前并不严格符合SGML标准。后来出现了HTML4，XHTML（符合XML标准的HTML），这两者都符合SGML的要求。

W3C在制定HTML5标准时，希望摆脱SGML的一些无用的功能和声明，并延续HTML4的风格，而非严格的XHTML的风格。

页面的渲染模式

页面的渲染模式根据是否声明了DOCTYPE分别触发兼容模式和标准模式。

document.compatMode可以返回当前页面的兼容模式：

document.compatMode
> CSS1Compat //标准模式
> BackCompat //怪异模式、兼容模式

标准模式页面排版和JS运行模式都是以浏览器支持的最高标准运行。兼容模式中，页面以宽松向后兼容的方式显示，模拟老式浏览器的行为已防止站点无法工作。
