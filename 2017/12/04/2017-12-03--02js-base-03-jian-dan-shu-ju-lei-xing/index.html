1.undefined

undefined只有一个值，就是undefined。字面上的意思是未定义。

比如我们设定变量，但没有赋值的时候，这个变量的值就是undefined。

补充：

  • 令人迷惑的一点是，对于未声明的变量执行typeof会返回undefined。比如：
var name;
console.log(typeof name);//undefined
console.log(typeof(age));//undefined,并没有声明age变量，但还是会返回undefined

2.null

null表示一个空对象指针。

  • 如果我们设定的变量会用来保存对象，那么最好将变量初始化为null。
  • undefined值是null的派生值，所以undefined==null返回值为true。

讲到这里呢，如果变量将来用来保存字符串，那么可以将变量初始化为""。

JS的设计者希望用null表示一个空的值，而undefined表示值未定义。实时证明，这并没有什么卵用。大多数情况下，我们都应该用null。undefined仅仅在判断函数参数是否传递的情况下才有用。

引自：《廖雪峰的JavaScript全栈教程》

3.number

数字有三种：整数、浮点数（小数）、NaN（Not a Number非数值）。

表示

​ 除了用十进制，还可以用八进制和十六进制的字面值来表示。

  • 其中八进制第一位必须是0，然后是八进制数字序列（0-7）。这样的话，在数值转换时，ES3和ES5会出现歧义。比如070在ES3下认为是八进制的56，在ES5下认为是十进制的70。所以，八进制字面量在严格模式下无效。
  • 十六进制的前两位必须是0x，后面跟任意十六进制的数字（0~F），字母可大写可小写。
  • 在进行算术运算时，所有其他进制表示的数值最终都会转换为十进制的数值。

数值范围

  • Number.MIN_VALUE：计算机保存的最小数值；Number.MAX_VALUE：计算机保存的最大值。
  • 数值超过最大值，用Infinity（正无穷大）、-Infinity（负无穷大）表示
  • isFinite()函数用来确定数值是否在最大和最小数值之间，返回布尔值。
  • 对于极大或极小的数值，可以用e表示法（科学计数法），e相当于10的指数次幂。比如：310=3.1e2。

浮点数的特点

  • 浮点数计算会有误差，这和浮点数的储存方式有关。比如：alert(0.07*100);//7.000000000000001。这一点在写透明度渐变的代码时，常需要Math.round()四舍五入浮点数。整数的计算精度会好很多。
  • ECMScript默认将小数点后带有6个零以上的浮点值转换为科学计数法表示。比如：0.0000003会被转换为3e-7。
  • 保存浮点数所需的内存空间是整数的两倍。

NaN

  • 任何一个涉及NaN的操作都会返回NaN
  • NaN与任何值都不相等，包括它本身
NaN的检测 说明
isNaN()函数 用来检测参数是否为“非数值”。返回值为true/false

用isNaN()检测也不是特别准确：

虽然，JavaScript提供了isNaN来检测某个值是否为NaN，但是，这也不太精确的，因为，在调用isNaN函数之前，本身就存在了一个隐式转换的过程，它会把那些原本不是NaN的值转换成NaN的，如下：

isNaN("foo");               // true
isNaN(undefined);           // true
isNaN({});                  // true
isNaN({ valueOf: "foo" });  // true

上面代码，我们使用isNaN来测试后，发现字符串，undefined，甚至对象，结果都返回真！！！但是，我们总不能说他们也是NaN吧？总而言之，得出的结论是：isNaN检测NaN并不可靠！！！

不过isNaN是否可靠这一点我个人认为还有待商榷。

数值转换：

数值转换方式 说明
Number()函数 强制将任何类型的数据转换为数字类型的数据。不能转换为数值的转换为NaN
parseInt()函数 将字符串转换为整数数值。
- parseInt()函数操作字符串可以忽略前面的空格或者前导零。
- parseInt()函数操作字符串必须以数字开头。
- parseInt()函数遇到第一个非数字或者点.转换结束
- parseInt()函数有两个参数，第一个为要转换的字符串，第二个为转换进制。
parseFloat()函数 将字符串转换为浮点数或整数。
- parseFloat()函数操作字符串可以忽略前面的空格或者前导零。
- parseFloat()函数操作字符串必须以数字开头。
- parseFloat()函数第一个非数字或者第二个点.转换结束。
- parseFloat()函数只有一个参数。
  • 前导零，比如：01203中的第一个零就是前导零。
  • parseInt()和parseFloat()专门用来把字符串转换为数值。不建议一上来就用Number()转换。
  • 注意：用不同的函数转换数据得到的结果不一定相同。详见：Number()、parseIn()和parseFloat()的区别

4.string

string类型表示由零或多个16位Unicode字符组成的字符序列，即字符串。用"或'表示。

  • ECMAScript中字符串的数据是不可改变的，也就是说，字符串一旦创建，它们的值就不能改变。要改变某个变量保存的字符串，首先要销毁原来的字符串，再用新字符串填充该变量。
  • 任何字符串的长度可以通过length属性取得。如果包含双字节字符，那么获得的长度可能不精确。

数值转换：

数据转换方式 说明
str.toString()方法 可以将除undefined和null之外的任意数据转换为字符串，可指定转换进制（2,8,10,16）
String()函数 可将所有类型的数据转换为字符串，包括undefined和null，不可指定进制

true和false也是可以转换的。

所以，对于不明确数据类型的，尽量用String()转换。

关于toString()可以指定转换进制，看下面的例子：

var num=12;
num.toString(2);  //'1100'(先把12转换为二进制数值，再转换为字符串，以下同理)
num.toString(8);  //'14'
num.toString(10); //'12'
num.toString(16); //'c'

关于str.toString()和String()还有更多要了解的地方，详见：js中toString()和String()区别详解

刚才犯傻，在想，如果要转换的不是数字str.toString()还能按转换进制吗？

答曰：这个指定转换进制的参数只对数字有效，对于非数字，本来就谈不上按进制转换，就算加上该参数，也不会影响转换。再转念一想，其实能转换成字符串的的就只有数字、boolean、undefined和null，所以，基本上大部分要转换的就是数字。

5.Boolean

数据转换方式 说明
Boolean()函数 将任意类型数据转换为布尔值。
  • 除0和NaN以外的所有数字都转换为true
  • 除空字符串""以外的所有字符串，包括空格字符串，都转换为true
  • null和undefined转换布尔型为false

总结起来，就是：0、NaN、空字符串、undefined、null都会转换为false。它们都是无、没有。

7.补充

typeof操作符

  • 用来检测变量类型
  • typeof是一个操作符，不是函数，尽管可以使用()，但不是必需的。
  • 常用typeof来检测简单数据类型，对于null会返回Object,因为null是空对象指针。
  • 对于对象类型数据，还分为很多其他对象类型，所以，常用instanceof()来检测对象的具体类型。
  • 函数本质上也是对象，但是函数也确实有一些特殊属性，所以，用typeof检测函数会返回function。
  • 对于正则表达式ExpReg，用typeof检测，在safari5及之前的版本、Chrome7及之前的版本会返回function，其他浏览器返回Object。
