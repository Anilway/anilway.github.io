对于函数最基础的理解参看：《廖雪峰JavaScript教程》

函数实质上也是对象。

  • 每一个函数都是Function对象的实例。
  • 函数名实际上就是函数对象的指针。所以，使用不带括号的函数名是访问函数的指针；使用带括号的函数名是调用函数。

1、创建函数

有三种方式：函数声明、函数表达式、使用Function构造函数。

函数声明：

function sum(num1,num2){
  return num1+num2;
}

函数表达式：

var sum=function(num1,num2){
    return num1+num2;
};

注意末尾的分号，相当于把函数赋值给变量sum。

使用Function构造函数：（其实也是函数表达式）

var sum=new Function("num1","num2","return num1+num2");

使用Function构造函数来创建函数并不实用，在实际编程时很少用到，在代码执行时有两次解析（第一次是常规代码解析，第二次解析传入构造函数的字符串），不过很容易让我们理解函数也是对象。注意，最后一个参数始终被看成函数体。

还要补充一种：自执行函数表达式

主要用于创建一个新的作用域，在此作用域内声明的变量不会和其它作用域内的变量冲突或混淆，大多是以匿名函数方式存在，且立即自动执行：

 (function () {
     // var x = ...
 })();

此种自执行函数表达式也算是函数表达式。

函数没有重载。在同一作用域下，如果有两个同名函数，后一个函数会覆盖前一个函数。（参看《JS高程》p66和p111）。如果是不同对象的同名方法倒是不会覆盖。

2、函数声明提升不就是执行环境准备的一部分么？

先看个例子：

alert(foo());//hi!
alert(foo1);//undefined
function foo(){
  return "hi！";
}
var foo1=function(){
    return "嗨！";
}

在上面的例子中，虽然函数写在alert后面，但是使用函数声明创建的函数可以正常调用，但是使用函数表达式创建的函数就不能被正常调用。

这就是函数声明提升。一般都解释说在代码执行前XXX被提升到作用域顶部。其实啊，就是执行环境的准备工作嘛！干嘛要搞的那么复杂。

在代码执行前，解释器（引擎）会先对：

  • 函数声明——赋值
  • this——赋值
  • 变量、函数表达式——默认赋值为undefined。（在执行的时候再赋值）
  • 参数——赋值
  • arguments——赋值
  • 自由变量的取值作用域——赋值

所以，什么函数声明提升、变量提升都是执行环境的准备嘛！

例子：来源：深入解析变量声明提升和函数声明提升

var say = function(){
  console.log('1');
};

function say(){
  console.log('2');
};

say(); //输出：'1'

解析：预编译阶段进行变量声明提升和函数声明提升后，上述代码执行效果等同于：

var say; //变量声明提升

function say(){ //函数声明提升
  console.log('2');
}

say = function(){ //变量赋值保持原位执行，say函数被覆盖
  console.log('1');
};

say(); //输出'1'

变量声明和函数声明提升的优先级

函数声明提升的优先级要高于变量声明提升。

先看一个简单示例：

console.log(say); //输出：[Function: say]

function say(){
  console.log('1');
};

var say = '2';

console.log(say); //输出'2'

解析：本例中声明的函数和变量同名都是say，且函数声明在先，变量声明在后，按理说第一次打印say值预期会是undefined，然而结果是[Function: say]。

预编译阶段进行变量声明提升和函数声明提升后，上述代码执行效果等同于：

var say = function (){ //函数声明（包括定义）提升
  console.log('1');
};

var say; //只是声明，并不会覆盖say的值

console.log(say); //故输出：[Function: say]

say = '2'; //此时say会被覆盖

console.log(say); //输出'2'

总结：同名情况下，函数声明提升优先级要高于变量声明提升，且提升后该函数声明定义不会被提升后的同名变量声明所覆盖，但是会被后续顺序执行的同名变量赋值所覆盖。

优先级没看明白？再看看下面的例子：

来源：变量和函数声明提升

函数声明提升的优先级是高于变量声明的，这里的优先级指的是：同名的变量和函数中，函数先提升，再遇到同名的变量或函数，提升都将被编译器忽略。

//同时声明变量a和函数a
var a;
function a() {} 
alert(typeof a); //显示的是"function"，初步证明function的优先级高于var。

//先声明函数后声明变量，证明上边的例子不是function覆盖了变量
function a() {}
var a; 
alert(typeof a); //显示的仍是"function",而不是"undefined"，即function的优先级高于var。

注意，上面只是说声明的情况，如果变量同时赋值，有不一样了

//声明了变量的同时赋值
function a() {}
var a = 1;
alert(typeof a); //number,此时不是function了。
//说明："var a=1"相当于"var a;a=1",即先声明，后赋值，"a=1"相当于把a重新赋值了，自然就是number!

我再补充一下，如果是：

var a = 1;
function a() {}
alert(typeof a);//number

函数声明提升优先级高于变量提升，意思是说，在执行环境准备阶段（预编译阶段），首先给a赋值为函数了，无论function在var前或者后面，然后在代码执行的时候，发现有代码var a=1,所以预编译阶段的a的值被覆盖为1，所以，alert(typeof a)为number。

啰嗦一下，在执行上下文，变量a不会被赋值，在执行阶段，才会按代码内容赋值。所以，就算函数声明优先级高，对于同名变量，如果有赋值代码，在执行时，函数声明会被后面的赋值覆盖。

3、函数的内部属性

在函数内部有两个特殊的对象：arguments和this。注意区别函数内部的属性和函数自己的属性和方法。

arguments：是特殊的数组，用来保存函数的参数。它是类数组，有length属性，没有数组的其他属性。不过，arguments也有自己的属性，callee属性。

callee属性：该属性是一个指针，指向arguments对应的函数。

至于this，以后会专门再讲，在此暂时不讲。

arguments.callee的常用的作用就是可以代替函数名作为函数的指针，避免函数名更换影响函数执行。你可以把arguments.callee理解为当前函数。

例如：阶乘函数(递归算法)

function sum(num){
  if(num<=1){
    return 1;
  }else{
    return num*sum(num-1);
  }
}

可是，如果我把函数名赋值给其他变量，就无法执行该函数了：

var foo=sum;
alert(foo(5));

所以，用arguments.callee就消除这种紧密耦合的现象：

function sum(num){
  if(num<=1){
    return 1;
  }else{
    return num*arguments.callee(num-1);
  }
}
var foo=sum;
alert(foo(5));

4、函数属性和方法

caller：保存调用该函数的函数的引用。

  • ES5中出现的，ES3中并没有该属性
  • 如果是在全局作用局中调用该函数，它的值为null

例如：

function outer(){
  inner();
}
function inner(){
  alert(inner.caller);//为了避免紧密耦合问题，可以写为alert(arguments.callee.caller)
}
outer();//警告框中显示outer()函数的源代码
        //即function outer(){inner();}

在这个例子中，调用inner()函数的是outer()函数，所以警告框中弹出的是ouert()函数的源代码。

注意和前面的callee区分开：

  • callee是函数内部arguments的属性，保存的是该函数的指针。arguments.callee相当于函数名
  • caller是函数的属性，保存的是引用该函数的函数的指针。函数名.caller=arguments.callee.caller

在ES5下还定义了arguments.caller属性。定义arguments.callee属性是为了分清arguments.caller和函数的caller属性。

函数所处模式 非严格模式下 严格模式下
arguments.callee 保存该函数指针 报错
arguments.caller undefined 报错，而且不能为函数的caller属性赋值

这样做是为了加强这门语言的安全性，这样第三方代码就不能在相同的环境里窥视其他代码了。？？？

函数是对象，所以每个函数都有属性和方法。每个函数都包含length和prototype两个属性。

length：函数希望接收命名参数的个数

可别误以为是arguments的属性哦！

  • 函数名.length：函数希望接收的命名参数的个数
  • arguments.length：函数实际接收的参数的个数

我们都知道函数是可以接受任意个参数的，参数都保存在arguments数组中。函数名.length表示我们写函数时，预设的参数个数。

例如：

function foo(num1,num2){
  alert(arguments.length);
  return num1+num2;
}
alert(foo.length);//2
alert(foo(1,2,3));//3

至于prototype涉及到原型、原型链之类的，会在后面的博文中详细解释，在此暂不写。

apply()和call()

每个函数都包含两个非继承而来的方法：apply()和call()

它们的作用相同，都是指定特殊的作用域来调用函数。

比如：

window.color='red';
var o={color:'blue'};
function showColor(){
    alert(this.color);
}

showColor();            //red
showColor.call(this);   //red
showColor.call(window); //red
showColor.call(o);      //blue

//apply()和call()的作用一样
showColor();             //red
showColor.apply(this);   //red
showColor.apply(window); //red
showColor.apply(o);      //blue

但是，apply()和call()的区别在于传递参数上：

  • apply()传递参数要写参数数组或者arguments
  • call()传递参数要分别写出参数

例如：

function sum(num1,num2){
  return num1+num2;
}

//用apply()为：apply(指定作用域，参数数组或者arguments对象)
function sum1(num1,num2){
  return sum.apply(this,arguments);//这里的arguments对象是sum1的参数数组，this指的是sum作用域
  //或者return sum.apply(this,[num1,num2]);
}

//用call()为：call(指定作用域，参数1,参数2，……)
function sum2(num1,num2){
  return sum.call(this,num1,num2);//this指的是sum作用域，因为在这里可以看成sum对象的call方法，上面同理
}

//可是在《JS高程》P117中说sum1在执行sum()函数时传入了this作为this值（因为是在全局作用域中调用的，所以传入的是window对象）。所以，上面的this实际上是window，而不是sum？？？

1、call，apply都属于Function.prototype的一个方法，它是JavaScript引擎内在实现的，因为属于Function.prototype，所以每个Function对象实例(就是每个方法)都有call，apply属性。既然作为方法的属性，那它们的使用就当然是针对方法的了，这两个方法是容易混淆的，因为它们的作用一样，只是使用方式不同。

2、语法：foo.call(this, arg1,arg2,arg3) == foo.apply(this, arguments) == this.foo(arg1, arg2, arg3);

3、相同点：两个方法产生的作用是完全一样的。

4、不同点：方法传递的参数不同。

来源：JavaScript中的apply()方法和call()方法使用介绍

bind()

bind() 函数会创建一个新函数（称为绑定函数），新函数与被调函数（绑定函数的目标函数）具有相同的函数体（在 ECMAScript 5 规范中内置的call属性）。当新函数被调用时 this 值绑定到 bind() 的第一个参数，该参数不能被重写。绑定函数被调用时，bind() 也接受预设的参数提供给原函数。一个绑定函数也能使用new操作符创建对象：这种行为就像把原函数当成构造器。提供的 this 值被忽略，同时调用时的参数被提供给模拟函数。

来源：Function.prototype.bind()

例如：

window.color='red';
var o={color:'blue'};

function sayColor(){
  alter(this.color);
}

var foo=sayColor.bind(o);
foo();//blue

上面的例子中，sayColor()为被调函数，通过使用bind()方法创建出一个新函数，变量foo作为指针接收这个新函数，即新函数为foo()，函数foo()即为绑定函数。bind()的第一个参数为指定的作用域（上下文），或者说是指定的this值，上面的例子中指定的为对象o，所以，调用foo()结果为blue。
