1、Apache安装

下载

下载地址：http://httpd.apache.org/

配置

  • SeverRoot：服务器所在的目录
  • DocumentRoot：网站默认根目录
  • Directory：访问权限
  • ServerName：服务器的名称

打开在Apache文件目录下conf下的httpd.conf:

Apache24\conf\httpd.conf

然后在大约第38行代码可以看到：

# Define SRVROOT "/Apache24"
# ServerRoot "${SRVROOT}"

# 上面的是默认的值，下面使我们修改后的

Define SRVROOT "D:/server/Apache24"
ServerRoot "D:/server/Apache24" # 这里配置服务器所在的目录，也就是Apache的安装目录，注意目录名不能是中文或出现空格

然后在大约248行代码可以看到：

# DocumentRoot "${SRVROOT}/htdocs"
# <Directory "${SRVROOT}/htdocs">

 DocumentRoot "D:/server/Web" # 我们在这里设置网站的根目录，最好不要设置在Apache里面，我们放在外面方便管理
 <Directory "D:/server/Web"> # 这里设置访问权限也是如此

在大约224行代码可以看到：

ServerName localhost:80 # www.exapmle.com

我们可以修改站点名称。

安装

我们要用管理员身份使用命令行，在Apache24\bin下调用httpd.exe进行安装。

我们使用命令：

httpd.exe -k install

进行安装，这个命令并且会将Apache安装成windows系统服务。这样的好处是，以后我们电脑开机了，就会自动启动该服务，省去了我们手动开启的过程。

我们在初次安装的时候，可能会因为电脑组件不完善出现下面的弹窗提示：

我们需要使用下面的工具进行修复：

下载：微软官网

  • x86： vc_redist.x86.exe （32位系统选择此文件）
  • x64： vc_redist.x64.exe （64位系统选择此文件）

在下载页面点击下载，转跳到文件列表，选择你需要的文件下载安装即可修复该问题。

然后我们再次安装，就安装成功了。

运行

接着我们来运行apache服务。

在命令行中输入命令：

httpd.exe -k start

即可开启服务。服务开启成功后，我们可以在浏览器输入localhost，就可以看到：

我们可以用下面的命令来检测服务器配置文件httpd.conf是否有语法错误：

httpd -t

每当我们修改服务器配置文件后，需要重启apache服务才会生效。

我们可以使用Apache检测器软件来进行重启，还可以进行开启或关闭。

这个程序位置在..\Apache24\bin目录下。

2、安装PHP

如果我们要运行一个动态网站，光靠Apache是不够的，还需要借助动态语言php才可以。

下载

下载地址（windows版）：https://windows.php.net/download#php-7.2

安装

将下载后的文件解压，放在server（就是你放Apache的那个文件夹）文件夹下。然后，可以将php的文件夹名称改成php7，方便管理。

我们可以简单看一下php文件夹中的内容：

配置

我们在使用php的时候，需要一个配置文件，php本身给我们提供了两种：

  • php.ini-development开发环境下的配置文件
  • php.ini-production生产环境下的配置文件

那我们在开发的时候就是用开发环境配置文件，就复制一份php.ini-development,并重命名为php.ini。注意这里ini是后缀名。

当我们以后网站上线的时候，就将配置文件换成生产环境配置文件即可。当然也需要重命名为php.ini。

测试php是否安装成功

我们在Web目录下创建一个index.php文件：

<?php  
    // 输出
    echo 'hello world';
?>

然后我们使用命令行，让php.exe调用我们创建的index.php文件，看能否正常返回。

在php7目录下运行命令行，输入命令

php.exe -f D:\server\Web\index.php

如果返回hello world，既表示php安装成功。

3、将PHP与Apache关联起来

Apache能够进行某些操作，就是因为Apache加载了某些模块。在我们目前的安装下，PHP与Apache都是独立运行的。

比如，我们现在访问http://localhost/index.php，页面显示如下：

php本身提供了一个php7apache2_4.dll的这样的一个文件，用来与Apache关联起来。我们只需要在Apachce中进行配置，让Apache能够加载这个模块即可。

如果你的php文件中没有php7apache2_4.dll文件，那是因为你下载的NOT THREAD SAFE版本的php，你需要下载 THREAD SAFE版本的php才会有。重新在这个页面查找下载https://windows.php.net/download#php-7.2。

配置如下：

在Apach的配置文件中添加模块。Apache24\conf\httpd.conf

大概在184行代码。

#加载php7 注意严格区分大小写
LoadModule php7_module D:/server/php7/php7apache2_4.dll

现在，我们可以检测一下httpd.conf有没有语法错误，还是在在Apache24\bin下打开命令行，输入httpd -t进行检测。

我们前面说了，修改了服务器配置文件，需要重启才会生效。但是到目前为止，我们重启服务器，还是不会生效，php还是没有和Apache关联起来。

因为我们目前只是让Apache加载php，还没有告诉php做什么事。所以，我们还要继续配置。

配置如下：

在Apach的配置文件中给php分配任务。Apache24\conf\httpd.conf

继续跟着上面的代码往后写：

#分配工作给php:所有以.php为后缀的文件都交给php.exe处理
AddType application/x-httpd-php .php

现在我们修改完配置文件后，重启Apache，再次访问：http://localhost/index.php，得到的结果就是：

到目前为止，我们的php和Apache就关联成功了。但是现在php虽然可以完成Apache分配的任务，但是php是独立运行的。要想让php完全服从Apache，还需要将php的配置文件加载到Apache中。我们继续在Apache24\conf\httpd.conf中添加以下代码：

#加载php配置文件 后面的php.ini可以不写
PHPIniDir "D:/server/php7"

这样，php和Apache的关联算是完成了。我们再次重启Apache，然后访问index.php还是ok的。

上面的配置总结起来就是：

现在我们再来做个测试，看一下php的运行环境。

我们把index.php的代码修改一下：

<?php  
    phpinfo();// php自带的函数，可以查看运行环境
?>

页面就会呈现很多相关的信息。

4、修改php开发时区

我们可以在php运行环境中看到默认的开发时区是UTC。

这个UTC是世界时区，东一区，就是以伦敦那个地方的时区为准。但是我们是在中国进行开发的，所以，要改成中国的时区。

这个要在php的配置文件（php.ini）中进行修改。

修改如下：

大概在代码932行。

;增加中国时区 分号表示注释
date.timezone=PRC

虽然我们修改的是php.ini，但修改后，也需要重启Apache才会生效。之后我们再来查看，就是中国时区了。

5、安装MySQL数据库

在前面我们已经让php和Apache进行了关联，可以进行动态网站的开发了。但是动态网站的很大一个特点就是信息交互，php作为一种程序语言，是没有办法长期保存数据的，按目前的配置，数据只能暂时保存在内存中。

为了能够长期保存数据，我们需要安装MySQL进行数据的持久化保存。

MySQL数据库是一个独立的软件，是一种在php需要的时候通过代码去操作的软件。因为MySQL的安装可以独立出来，不需要在安装时就与Apache或者php关联（Apache无法操作数据库）。同时MySQL也是一款免费开源的数据库产品，深受广大开发公司的喜爱。

下载

下载地址：https://dev.mysql.com/downloads/mysql/

在页面下面，需要往下滚动才能看到下载地址。

注意，我们要下载第一个文件192.3M的那个。

将下载后的文件解压，放在server目录下即可。为了方便管理，我们可以将MySQL文件夹名称改成mysql。

添加环境变量

因为我们以后会经常用到mysql/bin文件夹下的各种程序，这些都是要在命令行中调用的。为了方便每次打开命令行能直接使用，我们可以添加环境变量。

以win7为例：

找到计算机，右键点击，选择属性。然后弹出控制面板：

我们在控制面板的左上角选择高级系统设置，会弹出窗口：

选择环境变量。然后再弹出的窗口中，选择系统环境变量中的Path，选择编辑：

在弹出的窗口中添加变量：;D:\server\mysql\bin（分号是为了与前面隔开）

我们也可以加上Apach的：;D:\server\Apache24\bin。

然后一路确定保存即可。

添加mysql配置文件

我们需要手动创建一个mysql.ini文件，添加以下代码：

[mysqld]
port = 3306
basedir = D:/server/mysql
datadir = D:/server/mysql/data
max_connections=200
character-set-server=utf8
default-storage-engine=INNODB
sql_mode=NO_ENGINE_SUBSTITUTION,STRICT_TRANS_TABLES
[mysql]
default-character-set=utf8

含义如下：

关于参数的含义，可以查看：mysql配置文件参数详解

安装

以管理员的身份运行CMD，执行以下命令，安装mysql：

mysqld.exe -install

因为我们前面设置环境变量了，所以可以直接进行安装。不用在D:\server\mysql\bin目录下执行cmd。

如果在安装时提示找不到MSVCR120.dll，需要像在安装Apache时那样，安装需要的运行库。你可以在网上查找一下。

然后再重新执行命令安装mysql。

安装成功后，我们还需要进行初始化操作。

执行以下命令：

mysqld.exe --initialize-insecure --user=mysql

初始化过程还比较长，需要等待十几秒才能完成。命令行显示如下：

启动服务

安装完成后，我们需要手动启动mysql。执行一下下命令：

net start mysql

这个命令是个服务命令，是windows自己的，和mysql没有关系。

配置用户及密码

默认安装mysql有一个超级管理员用户root，但是当前没有密码，非常不安全，因此需要修改root用户的密码。

输入一下命令：

mysqladmin -u root -p password 后面接着是新密码

操作动图演示：

我们设置密码后回车，会提示让我们输入旧密码，因为旧密码没有，所以我们直接回车即可。它会提示说这样设置密码不安全。我们可以先不用管。

测试安装效果

mysql是一种C/S结构软件，因此需要使用客户端去访问服务端，mysql提供了自己的客户端mysql.exe，可以在命令行（CMD）下通过mysql.exe进行访问。

输入命令：

mysql.exe -u用户名 -h主机地址 -P端口 -p密码

这种方式是明文输入，不安全。动图演示：

我们还可以使用密文输入的方式。其实，我们的主机地址默认是localhost时可以不写，端口号默认为3306时也可以不写。

输入命令：

mysql -u用户名 -p
