1、简介

  • angularJS是基于JS开发的客户端应用框架，使我们更加快捷，简单的开发web应用。

  • 诞生于2009年，后被google收购，用在了很多项目中。

  • 适用于CRUD应用或者SPA单页面网站的开发。

angularJS重点是做前后端分离式的开发方式，所以适用于CRUD应用或者SPA单页面网站的开发。因为这些应用中前后端交互是比较频繁的，比如增删改查，前后盾数据的ajax操作。

angularJS不太适合做一些很酷的特效，游戏、可视化的图形界面的开发，它更适合前后端数据交互的方案。

特点

  • MVC模式
  • 模块系统
  • 指令系统
  • 依赖注入
  • 双向数据绑定

2、MVC模式

MVC就是model-view-controller.

我们看一个简单的例子：

<!DOCTYPE html>
<html ng-app='app'>
<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <script src="source\angular-1.6.9\angular.js"></script>
</head>
<body>
    <div ng-controller='Aaa'>
        <p>{{name}}</p>
    </div>
    <script>
        let app=angular.module('app',[]);
        app.controller('Aaa',function($scope){
            $scope.name='Anna';
        });
    </script>
</body>
</html>

从这个例子中，我们可以看到：

在双大括号中的name和$scope.name对应。name就不是字符串，更像是一个变量。我们就可以把它理解为变量。这类似于模板的方式。在angularJS系统融入了模板的概念。

模板的特点就是通过一个双大括号形成一个表达式，在这个表达式中我们写的值就可以看做一个变量，就跟我们在JS中写的变量是一样，它会被解析成对应的值。

这和vue是一样的。我认为双括号里面就相当于一个小的代码执行环境，因为里面可以做简单的代码运算，比如加减乘除。

如果我们想真正输出name，就给name加上引号即可，这样就会把name当做字符串输出出来。

$scope.name='Anna'就是数据，<p><p>就是显示到页面上的值。它们之间是通过控制器联系到一起的，也就是ng-controller='Aaa'来关联的。

这就是一种MVC的方式。MVC的特点就是让model、view、conroller三者进行分离。

  • model指的就是数据，就是$scope.name='Anna'部分
  • view值的是视图，就相当于<p><p>部分（因为我们会在页面上看到，这就是视图嘛！）
  • controller就是控制器，它就把数据和视图联系到一起了，那就是ng-controller='Aaa'部分

至于控制器是如何实现将数据对应到视图上的，我们可以不用关心，是源码实现的。在初学阶段我们不用过于关心。当你深入学习的时候，可以研究源码的实现方式。

3、作用域

在上面的例子中，我们可以看到，数据是挂载在$scope上的（作为$scope的属性），这样视图才能找到。如果我们写的是变量name（定义一个name变量），视图就找不到数据。

app.controller('Aaa',function($scope){
    $scope.name='Anna'; // 视图可以找到挂载在$scope上的数据
    let name='Bob'; // 这种方式视图就无法找到数据,这个name就是一个普通的变量，不是一个数据
});

所以==在angularJS中，数据必须挂载到$scope对象上面，我们的视图才能找到数据。==

局部作用域$scope

$scope实际上是一种作用域（局部作用域），它只在自己的控制器函数中起作用。对应到视图上，就是它只对使用该控制器的标签范围内起作用。

比如前面的例子中：

<div ng-controller="Aaa">
    <p>{{name}}</p>
</div>

那么，这个div标签内部就是控制器Aaa中的$scope对应的作用域。

如果我们在这个标签外部去取控制器Aaa中$scope上的数据，是取不到的。比如：

<div ng-controller="Aaa">

</div>
<p>{{name}}</p>

这样就取不到控制器Aaa中$scope上对应的name的数据，不起作用。这个JS中函数作用域的概念类似。

同样的，如果我再写一个控制器Bbb，那么这个控制器也有自己的作用域$scope。它也只对使用该控制器的标签起作用。Aaa和Bbb之间的作用域互不干扰。

比如：

<div ng-controller='Aaa'>
    <p>{{name}}</p>
</div>
<div ng-controller="Bbb">
    <p>{{name}}</p>
    <p>{{age}}</p>
</div>

<script>
    let app=angular.module('app',[]);

    // 控制器Aaa
    app.controller('Aaa',function($scope){
        $scope.name='Anna';
        $scope.age=20;
    });

    // 控制器Bbb
    app.controller('Bbb',function($scope){
        $scope.name='Bob';
    });
</script>

这个例子中，虽然都是，但是对应的作用域不一样，视图对应的数据也不一样。

而且两个平行的作用域之间是互不关扰的，比如上例中Aaa控制器的作用域挂载的age数据，使用Bbb控制的标签就取不到这个age数据。

全局作用域$rootScope

在上面的例子中，我们可以把age数据挂载在全局作用域下，这样局部作用域就能访问到。

<div ng-controller='Aaa'>
    <p>{{name}}</p>
</div>
<div ng-controller="Bbb">
    <p>{{name}}</p>
    <p>{{age}}</p>
</div>

<script>
    let app=angular.module('app',[]);

    // 控制器Aaa
    app.controller('Aaa',function($scope,$rootScope){
        $scope.name='Anna';
        $rootScope.age=20;
    });

    // 控制器Bbb
    app.controller('Bbb',function($scope){
        $scope.name='Bob';
    });
</script>

这样，虽然在Bbb控制器中没有设置age这个数据，使用Bbb控制器的div标签中还是可以使用挂载在全局作用域下的age数据。

当然啦，如果Bbb控制器中有自己私有的age数据，那么这个div标签中就会使用私有的age数据，而不是全局age数据。这和JS的作用域概念是一样的。

<div ng-controller="Bbb">
    <p>{{name}}</p>
    <p>{{age}}</p> <!--显示30-->
</div>
<script>
    app.controller('Bbb',function($scope,$rootScope){
        $scope.name='Bob';
        $rootScope.age=20;
        $scope.age=30;
    });
</script>

对于全局作用域$rootScope来说，不管你写在那个控制器中都可以，挂载在$rootScope中的数据就是全局可以访问的。

作用域嵌套

这个概念和JS中的作用域链是一样的。

具体从代码上来看，是使用控制器的标签进行嵌套，形成了作用域的嵌套。在写控制器JS代码的时候，还是单独写的，并不嵌套在一起。

比如：

<div ng-controller="Aaa">
    <div ng-controller="Bbb">
        <p>{{name}}</p>
    </div>
</div>

<script>
    let app=angular.module('app',[]);
    app.controller('Aaa',function($scope,$rootScope){
        $rootScope.name='全局';
        // $scope.name='Aaa';
    });
    app.controller('Bbb',function($scope){
        // $scope.name='Bbb';
    });
</script>

在这个例子中，可以看到，在标签关系上可以看到，Bbb的上级作用域是Aaa，Aaa的上级作用域是全局作用域。

使用Bbb控制器的div标签中的会先在自己的作用域（Bbb）中找name数据，如果没有，就像上级作用域（Aaa）查找，如果还是没有，就继续像上级作用域查找，一直找到全局作用域（$rootScope）为止。

4、依赖注入

对于前面的例子，我们可以发现，控制器的函数表达式（controller方法的第二个参数）中有参数$scope、$rootScope。

app.controller('Bbb',function($scope){
    $scope.name='asdf';
    alert(123); // 可以说明这个函数确实被内部调用了
});

我们知道，函数的形参是可以更改的，那么这里的$scope是否可以更改呢？

在angularJS中是不能更改这些形参的。因为angularJS用到了依赖注入，这也是AngularJS的核心。

这个函数表达式的执行不是我们去执行的，而是angular内部去执行它，angular内部在执行这个函数的时候，已经自动把参数依赖注入进去了，不需要我们自己去给函数写参数。

那么这些参数已经自己注入进去了，我们如果想要得到自己想要的参数，就是根据我们写的形参来决定的。假如我们写的形参是$scope，那么这个时候就注入了一个局部作用域的对象。

在angularJS中，这些注入的对象有一个统一的名字，叫做服务。一般服务都是以$开头的，后面加上一个名字。这些服务一般是用来实现一个特定功能，来服务于我们实际的开发的。

依赖注入最简单的例子如下：

什么是依赖注入呢，我的理解，简单点就是说我的东西我自己并不像来拿着，我想要我依赖的那个人来帮我拿着，当我需要的时候，他给我就行了。当然这只是简单的理解，还是用代码解释比较清楚一些。

这里有一个function，很简单。

var a = function(name){
    console.log(name);
}

我们调用它：

a('abc')；//abc

那么，就像我上面说的，我能不能自己不传参数呢，例如：

a();//undefined

如何才能实现让别人帮我们注入这个参数呢：

var inject = function(name,callback){  
    return function(){     
        callback(name);  
    }
}

像这样，我们在定义参数的时候这样传：

a = inject('abc',a)

我们再调用a方法：

a()；//abc

5、指令与双向数据绑定

指令的基本概念

我们在DOM标签中写的以ng-开头的属性就是angularJS的指令。指令是用来完成一些功能的，与前面讲到的服务是类似的。

ng-controller

我们在前面已经熟悉了，ng-controller的作用就是连接数据和视图的桥梁；

ng-app

ng-app的作用是初始化的指令，如果我们不写ng-app指令的话，angularJS是不会执行的，因为没有初始化，无法调用。

ng-app是不仅可以加在html标签上，也可以加在其他标签上，比如body标签。你也可以加在一个div标签上，那么这个时候angularJS只对局部标签起作用，在这个div标签之外不起作用。

<div ng-app='app' ng-controller="Bbb">
    <p>{{name}}</p>
</div>

这样对程序开发是有好处的，因为有时候我们可能只在局部用到了angularJS，在其他地方用的是其他的框架。那么，我们就可以把ng-app放在局部标签，让angularJS只对局部起作用。

一般我们会写在html标签当中，这样是针对整个页面对angularJS的调用。

双向数据绑定

双向数据绑定是一种MVVM的设计模式，这种模式的特点是，当模型发生改变的时候，会影响视图的变化；当视图变化的时候，也会让模型数据发生改变。

我们以定时器为例：

<div  ng-controller="Aaa">
    <p>{{name}}</p>
</div>

<Script>
    let app=angular.module('app',[]);
    app.controller('Aaa',function($scope,$timeout){
        $scope.name="Anna";
        // let timer=setTimeout(function(){
        //     $scope.name="Bob";
        //     clearTimeout(timer);
        // },1000);
        $timeout(function(){
            $scope.name="Bob";
        },1000);
    })
</Script>

这里要注意的是，我们如果直接使用延迟定时器来写，页面不会发生变化，因为直接使用原生定时器，就算数据改变了，页面也不会刷新。

所以，我们要使用angularJS中的$timeout服务来编写。$timeout和原生的定时器的用法是一样的。我们在使用$timeout的时候，需要先注入进来，也就是在控制器的函数中添加$timeout参数。

$apply

如果我们非要使用原生JS或者第三方库的时候，使页面数据刷新呢，就可以使用$scope.$apply方法。

还是上面的例子，我们修改一下：

let app=angular.module('app',[]);
app.controller('Aaa',['$scope',function($scope){
    $scope.name='Anna';
    setTimeout(function(){
        $scope.$apply(function(){
            $scope.name='Bob'
        })
    },2000)
}])

==$scope.$apply中的回调函数可以监听挂载在$scope上的数据变化，一般数据发生变化，就能影响到视图。==所以，$apply的好处是可以在第三方库或者原生JS中使用。

其实，我们在使用$timeout服务或者下面的$ng-click的时候，内部都会调用$scope.$apply来实现影响视图的变化。我们直接使用$scope.$apply当然也可以啦。

ng-click

<div ng-controller="Aaa" ng-click="show()">
    <p>{{name}}</p>
</div>

<script>
    let app=angular.module('app',[]);
    app.controller('Aaa',function($scope){
        $scope.name="Anna";
        $scope.show=function(){
            alert(this.name);
        }
    })
</script>

ng-click相当于给该标签元素添加一个点击事件，它的值就是事件绑定函数。这个函数挂载在$scope上，ng-click会自动在$scope上去找到函数。

这个ng-click后面双引号的区域更像一个小的执行环境，它会把里面的字符串代码转换成代码执行。

所以，我们不仅可以写执行函数，写其他的代码也可以执行，比如：

<div ng-controller="Aaa" ng-click="name='Bob'">
    <p>{{name}}</p>
</div>

点击标签的时候，name数据就会变成Bob。

当然，也不是所有的代码都会执行，比如写成：

<div ng-controller="Aaa" ng-click="alert(123)">
    <p>{{name}}</p>
</div>

点击页面上的标签，并不会弹出弹窗。这和内部的实现方式有关，在此入门笔记中不继续追究。

除了ng-click指令之外，关于事件的，都会有相应的指令与其对应。

ng-model

前面在讲双向数据绑定的时候，只讲了数据影响视图，现在来看看视图如何影响数据。我们需要用到ng-model指令。

直接看例子：

<div ng-controller="Aaa">
    <input type="text" ng-model='name'>
    <p>{{name}}</p>
</div>

<script>
    let app=angular.module('app',[]);
    app.controller('Aaa',function($scope){
        $scope.name="Anna";
    })
</script>

我们给input标签添加ng-model指令，让该指令和name数据绑定，当视图上name数据改变时，$scope上的name也发生改变。

这和vue中的双向数据绑定是类似的。

6、过滤器与数据监听

过滤器

过滤器主要是用来对我们的数据进行格式化处理的。比如：把数字用金额格式表示，日期的格式化表示。

示例：

<div ng-controller="Aaa">
    价格：<input type="text" ng-model='goods.price'><br>
    个数：<input type="text" ng-model='goods.num'><br>
    总额：<span>{{sum()| currency:'￥'}}</span><br>
</div>

<script>
    let app=angular.module('app',[]);
    app.controller('Aaa',function($scope){
        $scope.goods={
            price:10,
            num:1
        }
        $scope.sum(){
            // 注意不要用this代替$scope，因为在做其他操作时，sum的this不一定指$scope，比如在后面要讲的函数监听的时候
            return $scope.goods.price*$scope.goods.num;
        }
    })
</script>

这里我们就是用currency过滤器来把数字结果转化为金额格式展现。和vue中的过滤器是类似的。

angularJS中除了currency过滤器之外，还有很多其他的内置的过滤器。当这些内置的过滤器不满足我们的需求的时候，我们可以自定义过滤器。

自定义过滤器示例：

app.filter('myfilter',function(){
    return function(val){
        return '￥'+val;
    }
})

这就是一个简单的过滤器。要注意的是，我们的过滤函数是写在app.filter方法的第二个回调函数中返回位置上。

然后我们就可以使用这个过滤器了：

<span>{{sum() | myfilter}}</span>

对于自定义过滤器，这只是最简单的例子；还有很多自定义过滤器的方式，在本入门笔记中不详细究。(后文有详细介绍)

拓展阅读： angularjs-自定义过滤器示例

数据监听

数据监听用到$scope的$watch方法。

基本使用方式：

$scope.$watch('goods.price',function(){
    console.log('监听到！')
})

当goods.price数据发生变化，就能被监听到。

如果你想同时监听goods.price和goods.num，只需要监听goods即可，但是要添加第三个参数true，表示深度监听。

$scope.$watch('goods',function(){
    console.log('监听到！')
},true)

$watch不仅可以监听字符串，还可以监听函数：

$scope.$watch($scope.sum,function(newVal,oldVal){
    console.log(newVal,oldVal);    
});

无论是监听变量，还是监听函数，监听方法的回调函数可以接收两个参数，第一个参数是监听到变化后的值，第二个参数是变化之前的值。

购物金额案例演示

<div ng-controller="Aaa">
    价格：<input type="text" ng-model='goods.price'><br>
    个数：<input type="text" ng-model='goods.num'><br>
    总额：<span>{{sum()| currency:'￥'}}</span><br>
    邮费：<span>{{goods.postage |currency:'￥'}}</span><br>
    支付：<span>{{sum()+goods.postage | currency:'￥'}}</span>
</div>

<script>
    let app=angular.module('app',[]);
    app.controller('Aaa',function($scope){
        $scope.goods={
            price:10,
            num:1,
            postage:10
        }
        // 购买的总价
        $scope.sum=function(){
            return $scope.goods.price*$scope.goods.num
        }
        // 当总价超过100元，免邮费
        $scope.$watch($scope.sum,function(newVal,oldVal){
            return $scope.goods.postage=newVal>=100?0:10;
        });
    });
</script>

7、模块化

模块化开发的好处：

  • 可以减少变量之间的全局污染，因为它是一个封闭的模块化方式。
  • 模块之间的互相依赖，不需要手动进行依赖处理。

声明模块

现在我们来创建一个模块，语法如下

angular.module('模块名称',[依赖数组])

angular就相当于jQuery中的$,它调用module方法来创建模块。需要两个参数才能创建模块。

第一个参数就是这个模块的名称。

第二个参数是一个数组，它表示当前这个模块要依赖的其他模块。

我们在实际项目中会有很多模块，模块之间需要相互依赖穿插开发，需要依赖的话，就通过这个数组将需要依赖的模块写进来。如果不需要依赖任何一个模块，就只需要写一个空数组就行了。

当我们有多个模块的时候，我们需要将其中一个模块作为初始模块。这个初始模块的模块名要和ng-app的值对应，作为初始模块。一般为：

let app=angular.module('app',[]);

这个时候，这个app就是作为初始模块，可以被调用。

接着，我们可以给这个模块添加控制器：

app.controller('Aaa',function($scope){
    $scope.name='Aaa';
});

app.controller('Bbb',function($scope){
    $scope.name='Bbb';
});

压缩处理

如果我们对代码进行压缩处理，上面的代码会出现问题，比如：

app.controller('Aaa',function($scope){
    $scope.name='Aaa';
});

中$scope可能在压缩的时候，可能会被简化为$s。前面我们讲到过一点依赖注入的原理，$scope形参是不能改变的。如果代码压缩的时候$scope被改变了，局部作用域就找不到了，就会出现问题。

所以，为了解决这个问题，我们可以把控制器的回调函数写在数组里：

app.controller('Aaa',['$scope',function($scope){
    $scope.name='Aaa';
}])

这样的话，回调函数的第一个参数和字符串'$scope'对应，在压缩的时候，就算$scope被压缩成$s，它也代表$scope,不受影响。

所以，我们在写代码的时候，尽量写成这种方式，这样代码上线的时候，就不会因为代码压缩出现问题。

8、常用的工具方法

bind()

bind方法用来修改this指向。

angular.bind(this指向,函数,参数1,参数2,...)(参数a,参数b,...)

其实和原生JS中函数的bind方法类似。

示例：

function show(){
    alert(this);
}

angular.bind(document,show)();

默认情况下，函数show的this指向window。这个例子中，我们通过angular的bind方法将函数show的this指向document。而且这个bind方法和原生JS中函数的bind方法类似，不会立即调用函数，我们需要手动调用。

我们可以在bind函数中给函数传参：

function show(n1,n2,n3){
    console.log(n1,n2,n3);
    alert(this);
}

// 在bind方法中给show传参
angular.bind(document,show,1,2,3)();

// 也可以在调用show函数的时候传参
angular.bind(document,show)(1,2,3);

// 也可以两边同时传参，参数都是按顺序传递的
angular.bind(document,show,1,2)(3);

copy()

用来拷贝对象。

示例：

let a={name:"Anna",like:['dog','mango']};
let b={age:27};
let c=angular.copy(a);

// a深拷贝给c
console.log(c); // {name:"Anna",like:['dog','mango']}
console.log(c===a);           // false
console.log(c.like===a.like); // false

还是上面的例子，我们给copy添加多个参数：

let a={name:"Anna",like:['dog','mango']};
let b={age:27};
let c=angular.copy(a,b);

console.log(c);
console.log(b);
console.log(b===c); // true

console.log(c===a);           // false
console.log(c.like===a.like); // false

console.log(b===a);           // false
console.log(b.like===a.like); // false

a把所有的值覆盖给了b，也是深拷贝。

如果我们在给copy添加第三个参数，是不起作用的。

总结起来copy方法有两种拷贝方式，第一种是：

// 将a拷贝给c
let c=angular.copy(a);

第二种是：

// 将a拷贝给b，b完全被覆盖
angular.copy(a,b);

这两种可以同时使用：

// 将a拷贝给了c和b
let c=angular.copy(a,b);

实际上，拷贝过后，b和c是相等的。也就是说，copy方法将a拷贝的副本返回出来，如果copy方法有第二个形参，就将a副本赋值给第二个形参。

extend()

对象继承。主要和上面的copy方法对比。

示例：

let a={name:"Anna"};
let b={age:27,like:['dog','mango']};
let c=angular.extend(a,b);
console.log(a); // {name: "Anna", like: Array(2), age: 27}
console.log(b); // {age:27,like:['dog','mango']}
console.log(a.like===b.like); // true


console.log(c); // {name: "Anna", like: Array(2), age: 27}
console.log(c===a); // true

可以看到，a继承了b的属性。我猜类似于a[key]=b[key]的方式循环过来的。因为a.like===b.like返回true。

继承的结果也会返回出来，从c===a返回true可以看出。

我们还可以实验一下，如果b中有和a同名的属性，b中的属性会覆盖a中的属性。

let a={name:'Anna',sex: "girl"};
let b={name:'Bob',age:27};
angular.extend(a,b);
console.log(a); // {name: "Bob", sex: "girl", age: 27}

angular.extend(b, a, z);  // b继承a和z中的属性

equals()

判断两个值是否相等。

let a=1,b=1,c=2;
console.log(angular.equals(a,b)); // true
console.log(angular.equals(a,c)); // false

但是要注意的是，它的判断和原生的判断有所区别：

let a=NaN,b=NaN;
console.log(angular.equals(a,b)); // true

console.log(angular.equals([],[])); // true

源码：

function equals(o1, o2) {
  if (o1 === o2) return true;
  if (o1 === null || o2 === null) return false;
  // eslint-disable-next-line no-self-compare
  if (o1 !== o1 && o2 !== o2) return true; // NaN === NaN
  var t1 = typeof o1, t2 = typeof o2, length, key, keySet;
  if (t1 === t2 && t1 === 'object') {
    if (isArray(o1)) {
      if (!isArray(o2)) return false;
      if ((length = o1.length) === o2.length) {
        for (key = 0; key < length; key++) {
          if (!equals(o1[key], o2[key])) return false;
        }
        return true;
      }
    } else if (isDate(o1)) {
      if (!isDate(o2)) return false;
      return simpleCompare(o1.getTime(), o2.getTime());
    } else if (isRegExp(o1)) {
      if (!isRegExp(o2)) return false;
      return o1.toString() === o2.toString();
    } else {
      if (isScope(o1) || isScope(o2) || isWindow(o1) || isWindow(o2) ||
        isArray(o2) || isDate(o2) || isRegExp(o2)) return false;
      keySet = createMap();
      for (key in o1) {
        if (key.charAt(0) === '$' || isFunction(o1[key])) continue;
        if (!equals(o1[key], o2[key])) return false;
        keySet[key] = true;
      }
      for (key in o2) {
        if (!(key in keySet) &&
            key.charAt(0) !== '$' &&
            isDefined(o2[key]) &&
            !isFunction(o2[key])) return false;
      }
      return true;
    }
  }
  return false;
}

辅助函数identity/noop

angular.identity函数就是你传的是什么，返回的就是什么。

let str='hello';
console.log(angular.identity(str)); // hello

原理类似于：

function identity(str){
    return str;
}

angular.noop函数就是返回undefiend。

console.log(angular.noop());// undefined

原理类似于：

function noop(){
}

这两个函数都是用来进行一些默认操作的,防止程序出问题。比如：

function getResult(fn, input) {
    //其实我们可以直接这样写：
    //return fn(input); 但这样写有个问题，如果传入的fn参数不是一个function类型，会导致出错。
    //官方推荐的写法如下：
  return (fn || angular.identity)(input);//这里如果fn参数为空值，使用angular.identity，那么当传入的第一个参数不是function类型时，相当于第一个参数是function(input){return input}
};

另外一个：

var getResult = function (fn, val) {
    //这里如果直接写为
    //return fn(val); 
    //也不会出错，但如果传入的第一个参数为空，则会出错。
  return (fn || angular.noop)(val);//这样写，如果传入的值不是函数，也不会出错
};

9、run方法操作

我们在前面注册模块之后，就是直接来创建控制器，和页面元素关联。比如：

<div ng-controller="Aaa">
    <p>{{name}}</p>

    app.controller('Aaa',['$scope',function($scope){
        $scope.name='Anna';
    }])

我们也可以不创建控制器，直接使用run方法。

没有控制器和页面元素链接，把name解析成一个全局变量。

比如：

</div>

        $rootScope.name="Coco";
    }])
</script>

==run方法的作用相当于去初始化全局的数据，我们把数据挂载在$rootScope上，这样页面上的变量就可以直接在根作用域上找到数据，不需要通过控制器来关联。==

省略了控制器指令的环境。

但是要注意的是，我们用run方法，数据是挂载在全局作用域上的，不能挂载在局部作用域上，run方法没有局部作用域，所以写$scope是无效的。

如果我们要挂载在局部作用域，还是要使用控制器来进行。

10.4 自定义过滤器

自定义过滤器是通过angular.module下的filter方法来实现的。

示例：我们来实现一个首字母大写的过滤器firstUpper

app.filter('firstUpper',function(){
    return function(str,num){
        num=num||0;
        return str.substring(0,num).toUpperCase()+str.substring(num);
    }
})

编写的过滤器中，str表示要过滤的数据，后面的num就是传入的参数。

这个过滤器的意思是默认将字符串的第一位字母转为大写。如果传了要转换的字符个数，就把前面num个字符转为大写。

使用：

<p>{{'hello' | firstUpper:2}}</p>

在JS中同样可以使用：

$scope.name=$filter('firstUpper')('hello',3);
