String.prototype.myTrim=function myTrim(){
    return this.replace(/^\s+|\s+$/g,"");

5、Function函数

bind方法在IE6-8下不兼容。我们可以手写一个bind方法。

Function.prototype.myBind=function myBind(context){
    context=context||window;
    var outerArg=Array.prototype.slice.call(arguments,1);
    var _this=this;

    // 如果bind存在，还是执行bind
    if("bind" in Function.prototype){
        outerArg.unshift(context);
        return _this.bind.apply(_this,outerArg);
    }

    return function(){
        var innerArg=Array.prototype.slice.call(arguments);
        // 如果返回的函数有默认传参，就加在最后
        outerArg=outerArg.concat(innerArg);
        _this.apply(context,outerArg);
    }
};

// 使用示例
document.body.onclick=fn.myBind(obj,100,200);

8、DOM中的兼容性问题

节点部分

  • 对于文本节点，高版本浏览器会把==空格==、==换行==也当做文本节点。也就是说，除了IE6~8，其他高版本浏览器会把空格和换行当做文本节点

节点的遍历

也就是节点的关系部分。

  • childNodes。这个本身没有兼容问题，大家都支持。问题在于节点的获取上，childNodes本身会获取容器的所有一级子节点。但是，前面说了，高版本浏览器会把空格、换行也当做文本节点。所以，如果我们在写html结构的时候，标签之间有空格或者换行，用高版本浏览器返回的结果和IE6-8浏览器返回的结果不一样，可能IE6-8返回的只有元素节点，但是高版本浏览返回的有文本节点。这样就看起来有兼容问题。（所以，在有些前端教程里面，会说这个有兼容问题，比如：高版本浏览器会算元素节点和文本节点，IE6-8会只算元素节点。当然这种说法是错的）
  • children。这个是用来获取容器的元素节点的。但是在IE6-8下，会把注释节点当做元素节点。
  • firstElementChild、lastElementChild、previousElementSibling、nextElementSibling，这些在IE6-8下都不兼容。我们可以自己写兼容的方法。

节点查找部分

  • document.getElementById。这个本身没有兼容性问题，但是我们用通过ID来获取元素的时候，==在IE7及更低版本==浏览器中，会把表单元素的name值当做id来识别使用（项目中尽量不要让表单的name和其他元素的id相同）
  • document.querySelector、document.querySelectorAll不兼容IE6-8
  • [context].getElementsByClassName在IE6~8浏览器中是不兼容的。在真实项目中如果要求兼容，需要自己实现一个兼容的方法。

  • document.getElementsByName这个本身没有兼容问题。就是对于IE浏览器来讲，只能识别表单元素的name属性值，所以我们这个方法一般都是用来操作表单元素的。

  • getElementsByClassName在IE6~8浏览器中是不兼容的。可以自己实现一个兼容的方法。

  • querySelector && querySelectorAll在IE6~8下不兼容，而且也没什么特别好的办法处理它的兼容（写兼容性相当于写一个jQuery的选择器了）。所以这两个方法一般多用于移动端开发使用（不过看资料上说，这两个方法的性能没有byID之类的好）

JS盒子模型部分

offsetTop、offsetLeft

在求偏移量的时候。在标准浏览器中，offsetLeft和offsetTop是当前元素的==外==边框到offsetParent元素的==内==边框的距离。但是对于==标准的IE8浏览器==来说，是当前元素的==外==边框到offsetParent元素的==外==边框的距离。

属性样式部分

  • getComputedStyle适用于一般浏览器，不兼容IE6-8；currentStyle用于IE。

    // 会带单位
function getStyle(ele,prop){
    if(ele.currentStyle){
        return ele.currsentStyle[prop];
    }else{
        return getComputedStyle(ele,false)[prop];
    }
}

DOM事件部分

DOM2级事件绑定

DOM2级事件绑定允许对同一元素的同一事件重复绑定方法，但是标准浏览器和IE低版本浏览器有所区别。

标准浏览器 IE6-8浏览器
[元素].addEventListenter（’事件名’，fun，false） [元素].attachEvent(==on==+”事件名”,fun)
this为当前元素，按顺序执行绑定，会忽略重复的绑定方法 this为window，不按顺序执行绑定，不会忽略重复绑定的方法（this问题、执行顺序问题、绑定去重问题）
可以直接在事件绑定的时候设定是否捕获，默认为false 需要通过ele.setCapture()来设置捕获，ele.releaseCapture()取消捕获。

我们可以自己写一个兼容的DOM2级事件绑定方法，通过自定义事件池来解决IE低版本浏览器下，DOM2级事件绑定的this问题、执行顺序问题和绑定去重问题。

MouseEvent事件对象

标准浏览器 IE6-8浏览器
传入事件对象e window.event。在DOM2级事件绑定attachEvent中，也是传递事件对象，但是这个事件对象和标准浏览器传入的事件对象，在属性上有所不同。
e.target（其实在标准浏览器的事件对象中，也有e.srcElement这个属性，我想大家不用它，是因为target比较好写吧） e.srcElement
e.pageX、e.pageY IE浏览器没有这两个属性，一般是用clientX+scrollLeft和clientY+scrollTop来兼容代替
e.preventDefault()阻止默认行为 e.returnValue=false
e.stopPropagation()阻止冒泡 e.cancelBubble=true

可以写一个兼容方法：

function ieMouseEvent(e){
    e=e||window.e;
    e.target=e.srcElement;
    e.pageX=e.clientX+winbox("scrolllLeft"); // winbox是自己写的JS工具库中的方法
    e.pageY=e.clientY+winbox("scrollTop");
    e.preventDefault=function(){
        e.returnValue=false;
    };
    e.stopPropagetion=function(){
        e.cancelBubble=true;
    };
    return e; // 也可以不用return
}

// 补充一下winbox方法

function winbox(attr,value){
    if(value){
        document.documentElement[attr]=value;
        document.body[attr]=value;
        return;
    }
    return document.documentElement[attr] || document.body[attr];
}

KeyboardEvent事件对象

对于键盘事件对象，IE6-8下没有code、key、witch属性。（key和code都是用来存储当前的键盘按键的，witch和keyCode一样，用来存储键盘的编码值）

但是好在标准浏览器和IE6-8浏览器都有keyCode属性。所以我们一般对于键盘事件都是用的keyCode。

事件传播

顺便说说mouseover和mouseenter的区别。虽然不存在什么兼容问题。

mouseover mouseenter
鼠标必须在==元素上==才触发该事件。如果该元素（假设叫ele）内部有子元素挡住了一部分，而恰好鼠标在子元素上，这个时候就算鼠标是在ele元素内部，但是没有ele元素上，就不能触发该事件 只要鼠标在该元素==区域内==，就触发该元素
有冒泡 会阻止冒泡传播
