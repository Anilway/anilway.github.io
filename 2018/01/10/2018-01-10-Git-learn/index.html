0.什么是仓库（Repository）

仓库是什么东西啊？项目是什么东西啊？在git中，把项目叫做仓库，其实本质上就是文件夹，就是目录。

1.配置用户

//配置
git config --global user.name "君の名は"
git config --global user.email "你的E-Mail"
//查看配置
git config --global list
git config -l //用这个命令也可以

2.创建仓库

方法一：把系统文件夹作为仓库

//进入文件夹，然后调出git命令行，输入命令
git init

然后Git就会自动把该文件夹作为仓库。

方法二：直接创建仓库

//在你想要创建仓库的位置，调出git命令行，输入命令
git init [name] //就会一个初始化空的仓库

方法三：远程拉取：

可以直接把网上的仓库拉回到本地当做本地仓库，比如github上面的。

//在你想要创建仓库的位置，调出git命令行，输入命令
git clone [url]
//如果你想重命名仓库，可在上面的命令后面直接加上你希望的名称
git clone [url] [name]

比如克隆这个仓库：https://github.com/Anilway/doc_trans。

//在clone or download按钮下复制git克隆地址，然后把该仓库拉回本地
git clone https://github.com/Anilway/doc_trans.git
//我希望仓库名称为Doc，那么，可输入命令
git clone https://github.com/Anilway/doc_trans.git Doc

3.基本用法

//【查看仓库目录】在该仓库文件夹根目录下输入命令：
ls -la
//【查看仓库状态】可以查看branch marster（分支）和commit（提交，就是后悔药，历史节点）
git status

git add .      // 把所有文件放到暂存区
git add [文件] // 把具体的某文件保存到暂存区

//【提交版本】
git commit -m "版本描述"

//【查看版本记录】
git log             // 如果看完了，按下q就退回来了
git log -p          // 查看该节点具体修改了啥，可以配合前面的代码使用（-p就是push）
git log --oneline   // 单行查看版本

//【退回到历史节点】
git checkout [节点Id]  // 回到指定节点
git checkout -         // 回到该版本的下一个新版本

注意，你提交的版本只提交你保存在暂存区的文件内容。如果你把修改的文件git add .到暂存去后，又对文件进行一次修改，那么你接着提交的版本中，不会有后面这次修改。所以，一定要把所有的修改都提交到暂存区之后，再进行版本提交。

4.文件的三种状态

  • modified 文件已修改
  • staged 已暂存
  • committed 已提交

5.版本标签

有时候，对于重要的版本，比如已经发布的完成本，V1.0 V2.0这种，就需要对这种重要的大版本添加标签。

//【添加标签】
git tag -a [标签名] -m "备注"           // 为当前版本添加标签；-a表示annotated有注释的
git tag -a [标签名] -m "备注" [版本id]  // 为某具体版本添加标签
git tag [标签名]                       // 如果没有-a，就是简单的tag，没有描述信息

//【查看标签】
git tag           // 查看所有标签
git tag [标签名]  // 查看某标签详细信息

6.分支

master//默认分支。master只是默认分支的名字而已，不是什么关键词，只是大家懒得改。

//【创建分支】
git branch [分支名]
git checkout -b [分支名] //创建分支并且换至该分支
//【查看当前在哪个分支】：可以看到head现在指向哪个分支
git log --all
git log --all --oneline //精简显示
git log --all --graph   //图示全部历史记录

//【切换分支】
git checkout [分支名]

//【合并分支】
git merge [分支名] //把该分支和当前分支合并
//如果合并的内容有冲突，你只要对冲突的内容进行更改，再提交就可以了

7.远程仓库

//【添加远程仓库】
git remote add [远程名称] [远程地址]

//【查看远程仓库】
git remote //列出所有的远程仓库
git remote -v //列出所有远程仓库的详细信息，会出现：
                 [远程名称][下载地址](fetch)
                [远程名称][上传地址](push)

//【上传代码】
git push -u [远程名][分支名]

如果你的本地仓库丢失，那么可以从远程仓库再clone回来。不过这个时候的远程名称默认变为origin了，不是自己之前设定的，所以在输入命令的时候要注意。

8.多人远程合作

//【获取远程更新】
git pull //相当于git fetch && git merge

//【节点冲突】
//如果多人同时修改，上传有版本冲突，那么后提交一方会被驳回，那么你只要再获取远程更新，整合代码，更新版本，再提交就可以了。

9.其他命令

cd            // 进入用户主目录；change directory
cd [下级目录]  // 进入下级目录
cd ~          // 进入用户主目录；
cd -          // 返回进入此目录之前所在的目录；
cd ..         // 返回上级目录（若当前目录为“/“，则执行完后还在“/"；".."为上级目录的意思）；
cd ../..      // 返回上两级目录；
cd !$         // 把上个命令的参数作为cd参数使用。

ctrl+l        // 清屏，这个是按键盘，不是命令哦！  

git --version // 查看Git版本，常用来判断Git是否安装成功

//连接两条命令&&
git add . && git commit -m "完成第一章"

//创建文件 
touch [文件名.扩展名]  //比如：touch test.text
