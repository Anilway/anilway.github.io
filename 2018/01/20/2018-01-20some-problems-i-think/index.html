1、深拷贝和浅拷贝

思考：如何把一个对象或者一个数组进行深度克隆？

自己实现的一个简单的深拷贝函数，可能考虑的还不太完全：

比如，只考虑了数组和普通对象类型的属性值的深拷贝。函数、正则就没有考虑了。

只拷贝了对象的私有属性。如果对象不是普通对象，而是一个构造函数的话，原型上的属性没有拷贝。

function deepCopy(obj){
    var ele={};
    for(key in obj){
        if(obj.hasOwnProperty(key)){
            if(Object.prototype.toString.call(obj[key])==='[object Object]'||Object.prototype.toString.call(obj[key])==='[object Array]'){
                ele[key]=deepCopy(obj[key]);
            }
            ele[key]=obj[key];
        }
    }
    return ele;
}

还有一种简单的深拷贝方法：targetObj=JSON.parse(JSON.stringify(copyObj));

这种方法有两个问题：

  1. 如果你的对象里有函数,函数无法被拷贝下来
  2. 无法拷贝copyObj对象原型链上的属性和方法

参考文章：《一篇文章彻底说清JS的深拷贝/浅拷贝》（说的不是特别好，基本上我自己都考虑到）

因为上面没有对函数进行拷贝。所以我在想，如何对函数进行拷贝。可能的思路有把函数代码转换为字符串，再把字符串转换为函数。但是，函数还有自己的原型，原型上还有属性。普通函数并不是构造函数，我认为在拷贝的时候也要把普通函数当做构造函数来考虑。因为它有原型。函数的原型要不要深拷贝呢？

函数的三种角色：普通函数、构造函数、对象。

对于函数的深拷贝，可以看成是如何对构造函数进行深拷贝。

  • 这一篇文章似乎写的不错：《在js中的深复制实现方法》

  • 还有这一篇，分析了第三方库的深拷贝源码：《深入剖析 JavaScript 的深复制》

2、函数参数的解构赋值

函数参数的解构赋值本质上还是对数组和对象的解构赋值。

我们知道函数的参数可以传递无数个。当我们给函数传递一个数组或者一个对象，才可以实现参数的解构赋值。从函数的角度来说，我们只给函数设置了一个形参，这个形参是数组或者对象。

function fn([a,b,c]){...}
function fn1({name,age}){...}

整体上看，这就是只给函数设置了一个形参。但是，从解构赋值的角度来看，这个参数里面实际上包含了多个形参。

当我们给函数传递形参的时候，传递的参数就是我们的目标数组（或目标对象）。

function fn([a,b,c=0]){
    console.log(a,b,c);
}
fn([12,23,34]);

function fn1({name:a,age}){
    console.log(a,age);
}
fn1({name:'Anna',age:9});

从解构赋值的角度看，形参在左边，传递的值在右边，进行解构赋值：

let [a,b,c=0]=[12,23,34];
let {name:a,age}={name:'Anna',age:9};

从而设定了变量了，并对变量赋值，在函数体中执行。当然，从函数的角度来说，这些是形参，是私有变量。

这就是我理解的函数的参数的结构赋值。

然后，再来看一看可能会和函数参数解构赋值混淆的地方。

函数本身就可以设定默认值

解构赋值是可以设定默认值的，而函数的参数本身也可以设定默认值。函数的参数本身设定默认值并不是解构赋值：

function fn(a=0){ // 如果不传参数，a就默认等于0
   return a;
}

fn();    // 0
fn(10);  // 10

不传参，就相当于传递的形参是undefined。所以，实际上是==undefined触发了形参的默认值==（在阮一峰《ECMAScript 6 入门》也有讲到）。所以，其实，如果我们传递的值是undefined，形参就等于默认值。

function fn(a,b=0,c){
    console.log(a,b,c); // 12 0 2
}
fn(12,undefined,2);

前面讲了，函数的参数能进行解构赋值，是因为只设定了一个形参，这个形参是一个数组或者一个对象。如果我们在执行函数的时候不传参，就会出现问题：

function fn([a,b,c=0]){
    console.log(a,b,c);
}
fn(); // Uncaught TypeError: Cannot read property 'Symbol(Symbol.iterator)' of undefined

function fn1({name:a,age}){
    console.log(a,age);
}
fn1(); // Uncaught TypeError: Cannot destructure property `name` of 'undefined' or 'null'.

这就相当于：

let [a,b,c=0]=undefined;
let {name:a,age}=undefined;

undefined没有这些属性啊，所以会报错。

为了避免这样的问题，我们从函数的参数的角度来看，可以给这个解构形参设定一个默认参数来避免这样的问题，比如：

function fn([a,b,c=0]=[]){
    console.log(a,b,c); // undefined undefined 0
}
fn();

function fn1({name:a,age}={}){
    console.log(a,age); // undefined undefined
}
fn1();

这样，如果我们没有传递参数，解构赋值还有默认的解构对象可以解构，避免报错。

运用：

let animate=function({curEle=null,target={},duration=1000,callBack=null}={}){
    ...
};
animate({
    curEle:box,
    target:{
        width:100,
        opacity:0.5
    }
});

既然我们可以给解构赋值设定默认解构对象。我们换一种思路，可以改变传递参数的结构：

function fn({x,y}={x:0,y:0}){
    return [x,y];
}
fn(); // [0,0]
fn({x:10,y:20});  // [10,20]

传递进去的是一个对象，返回出来的是对象的值组成的数组。就看你怎么运用了。

既然返回了数组或者对象，我们可以用解构的方式，接收函数返回的多个参数啊

function fn({x,y}={x:0,y:0}){
    return [x,y];
}
let [width,height]=fn({x:10,y:20});  
console.log(width,height); // 10 20

函数本身就可以使用扩展运算符

在数组结构中，我们可以使用扩展运算符：

let [a,...ary]=[12,23,34]; // a=12; ary=[23,34];
let {name,...arg}={name:'Anna',age:9,sex:'girl'}; // name='Anna';  arg={age:9,sex:'girl'}

而函数本身也是可以使用扩展运算符的。

function fn(...arg){
    console.log(arg); // [10, 20, "abc"]
}
fn(10,20,'abc');

这一点在箭头函数中运用尤其多，因为箭头函数中没有arguments，所以只能用这种方式来代替argumetns。这种方式反而更方便，因为获取的结果是一个数组，我们就可以很方便的直接使用数组原型上的方法。而argumetns是类数组，要使用数组上的方法，还需要进行转换。

function fn(){
    var ary=Array.prototype.slice.call(arguments,2);
    return ary;
}
fn('a','b',12,23,34,45); // [12,23,34,45]

/*------------------------------------------*/
function fn1(a,b,...arg){
    return arg;
}
fn1('a','b',12,23,34,45); // [12,23,34,45]

拓展一下，展开运算符还可以这么用：合并数组和对象

let ary1=[12,23,34],
    ary2=[45,56];
let ary=[...ary1,...ary2]; // [12,23,34,45,56]

let school = {name:'zfpx',age:9},
    me = {name:'jw',address:'回龙观'};
let obj = {...school,...me}; // {name: "jw", age: 9, address: "回龙观"}

前面使用的...描述有混淆的地方，它有两个叫法：展开运算符、剩余运算符。

区分方式：参考《ES6中扩展运算符(spread)和剩余运算符(rest)详解》。

不知道是否正确！

简单来说，就是：

当...放在形参（被赋值的一方）上，为剩余运算符；

当...放在实参（赋值的一方）上，为展开运算符；

// 放在实参里面，是展开运算符：
let ary=[12,23,34];
function sum(a,b,c){
    return a+b+c;
}
sum(...ary); // 这里就是展开运算符，把数组中的每一项展开

// 放在形参里面，是剩余运算符
let fn=(a,...b)=>{
    console.log(b);//  [20, 30, 40]
    console.log(b instanceof Array); // true
}
fn(10,20,30,40);

3、回调函数的上级作用域

let a='outer';
function fn(callback){
    let a='inner';
    callback || callback(a);
}

fn(function(value){
    console.log(a); // 'inner'
});

我理解的回调函数的上级作用域就是使用这个回调函数的函数的作用域。

函数的上级作用域就是定义函数的地方。函数是在哪个作用域定义的，函数上级作用域就是这个作用域。

  1. 函数执行时才会产生一个作用域出来。
  2. 函数虽然是存储在堆内存中，但是前提是，首先得有一个宿主环境才行，这个宿主环境就是作用域了，在全局就是群居作用域，在函数中就是私有作用域。

我们先抛开回调函数，先看看函数是如何定义的。在全局作用域中很简单，随便在最外面写一个函数就是。我们来看看在是由作用域下是如何定义函数的：

var a='outer';
function fn(){
    var a='inner';
    function inner(){
        console.log(a);
    }
}

上面的代码中，我们很容易的看出来，inner是在fn中定义的。

但是要注意的是，在目前这种请况下，inner还没有定义。最多是在全局作用域下，函数fn进行了函数声明提升。函数内部的代码以字符串的形式存储在堆内存中。

只有当fn执行的时候，inner函数才被创建出来。

执行fn，开辟了一个私有作用域，在私有作用域中，inner函数被声明，进行函数声明提升，开辟了一个堆内存在存储inner函数的代码。

这就是inner函数被定义的过程。

我们接着来看看回调函数，还是以开篇的例子来看：

let a='outer';
function fn(callback){
    let a='inner';
    callback || callback(a);
}

fn(function(value){
    console.log(value); // 'inner'
});

函数fn没有执行的时候，回调函数没有被定义，只有当fn执行的时候，产生一个私有作用域，回调函数才被声明定义出来。所以，根据此分析，我认为回调函数的上级作用域就是把该回调函数作为实参的那个函数。

我还认为，回调函数作为参数传给当前的执行函数，这个回调函数就是当前函数的私有变量值。

let a='outer';
function fn(callback){
    let a='inner';
    callback || callback(a); // 这里写错了，不需要把里面的a传进来，我们本身就要测试回调函数的上级作用域。
}

fn(function(value){
    console.log(value); // 'inner'
});

这就相当于在该执行函数的栈内存（作用域中定义了这个函数）。

// 执行函数的作用域中
1、形参赋值
callback=function(value){...};
2、变量提升
3、执行代码
4、栈内存是否销毁

如何查找上级作用域：

我们从一道题开始：

var n = 10;
function sum(){
    console.log(n);
}
sum(); // 10

/*--------------*/
~function (){
    var n=100;
    sum(); // 输出多少？ 还是10 sum的宿主环境是当前自治性函数形成的私有作用域。也就是说sum执行的时候是在当前私有作用域中形成的小作用域
}();

==函数执行形成一个私有作用域（A），A的上级作用于是谁，和他在哪儿执行的没关系，主要看它是在哪儿定义的。在哪个作用域下定义的，当前A的上级作用域就是谁。==

因为函数是引用类型啊，无论它在哪儿执行，它都需要回到原有存储空间去取代码，所以，函数就和它在哪个作用域下定义相关联。所以，它的上级作用域，就是它在哪个作用域下定义的这个作用域。

你可以理解为上级作用域就是父级，父级作用域就是它爹。它在哪儿定义的，谁把它生下来的，谁就是它爹，就是它的上级作用域。

鉴于上面复习的对上级作用域的定义。我还是有些疑问这个结论的，我们看看在函数中使用定时器的例子：

var a='outer';
function fn(){
    var a='inner';
    setTimeout(function(){
        console.log(a);
    },1000);
}

fn(); // 一秒钟后输出'inner'

在这个例子中，按照我上面的结论，定时器中的回调函数的上级作用域就是这个定时器，回调函数中没有a，就像上级作用域查找，定时器中有没有，就再想上级作用域查找。在此处定时器的上级作用域是fn吗？

定时器是window对象的一个方法。
