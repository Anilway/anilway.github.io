1、CSS盒子模型

传统CSS盒子模型

width、height：不是盒子的宽高，而是盒子中内容的宽度和高度。

盒子的宽度：

width+padding(Left/Right)+border(Left/Prght)

CSS3新增加的盒子模型

.box{
    box-sizing:border-box;
}

width和height不仅仅是内容的宽度，而是代表整个盒子的宽度（已经包含了padding和border），以后修改的padding或者border的值，盒子大小是不变的，内容宽高会随着改变。

2、JS中的盒子模型

提供一些属性和方法，用来描述当前盒子的样式。

cientWidth/clientHeight

当前盒子可视区的宽度和高度。

可视区域：内容的宽高+padding

clientWidth = width + padding(Left/Right)
clientHeight = height + padding(Top/Bpttom)

==和内容是否溢出，以及我们是否设置了overflow：hidden没有关系。==

比如下图的示例，文字超过内容大小，在padding区域还可以显示，超过padding就不能显示了。所以可视区域是内容的宽高+padding

在真实项目中，我们常会使用：

// 获取当前页面一屏幕的宽度（加或者是为了兼容所有的浏览器）
document.documentElement.clientWidth || document.body.clientWidth;

// 获取当前页面一屏幕的高度（加或者是为了兼容所有的浏览器）
document.documentElement.clientHeight || document.body.clientHeight;

面试题：

让一个盒子在整个页面水平和垂直都处于居中的位置？如果不知道盒子的宽度和高度如何处理？

用CSS来解决：

/*   
    使用定位：需要知道盒子的具体宽高   
*/ 
.box{
    position:absolute;
    top:50%;
    left:50%;
    margin:-50px 0 0 -50px;
    width:200px;
    height:200px;
    background: red;
}

/*  
    使用定位：不需要知道盒子具体的宽高（不兼容IE低版本浏览器）
*/
.box{
    position:absolute;
    top:0;
    left:0;
    bottom:0;
    right:0;
    margin:auto;
    background: red;
}

/*
    使用CSS3的flex伸缩盒子模型（不兼容）
*/
html{
    height:100%;
}

body{
    display:flex;
    height:100%;
    flex-flow:row wrap;
    align-items:center;
    justify-content:center;
}

.box{
    width:200px;
    height:100px;
    background:red;
}

使用JS解决：

// JS中只要获取到当前盒子具体的left和top值即可
/*
一屏幕的宽高 - 盒子的宽高，最后除以2，获取的值就是它应该具备的left、top（这个值可以让它处于页面中间）
*/


var winW = document.documentElement.clientWidth || document.body.clientWidth;
var winH = document.documentElement.clientHeight || document.body.clientHeight;
var boxW = box.clientWidth;  // 用offsetWidth更准确
var boxH = box.clientHeight; // 用offsetheight更准确

box.style.left=(winW-boxW)/2 + 'px';
box.style.top=(winH-boxH)/2 + 'px';

clientTop和clientLeft

只有top和left，没有right和bottom这两个属性。

clientTop：盒子上边框的高度

clientLeft：左边框的宽度

获取的结果其实就是border-width值。

==通过JS盒子模型属性获取的结果是不带单位的，而且只能是整数（它会自动四舍五入）==

.box{
    border:10.8px solid red;
}

box.clientLeft // 11

offsetWidth和offsetHeight

在clientWidth和clientHeight的基础上加上盒子的边框即可。

和内容是否溢出没关系。

真实项目中，如果想获取一个盒子的宽度和高度，我们一般用offsetWidth（而不是clientWidth），border也算是当前盒子的一部分。

scrollWidth和scrollHeight

没有内容溢出

获取的结果和clientWidth、clientHeight是一样的

有内容溢出

真实内容的宽度或者高度（包含溢出的内容），再==加上左padding==或者==上padding==的值（右padding或者下padding已经被溢出的内容占住了，所以不需要加）

==有内容溢出的情况下，我们通过scrollWidth或者scrollHeight获取的结果是一个约等于的值==

  1. 有内容溢出，每个浏览器由于对行高或者对文字的渲染不一样，获取的结果也是不一样的
  2. 是否设置overflow：hidden对最后获取的结果也有影响

我们经常用这个属性来获取当前页面的真实高度：

// 获取当前页面的真实高度(包含溢出的内容)
var winH = document.documentElement.scrollHeight || document.body.scrollHeight;

父级参照物及盒子的偏移量

offsetParent

offsetParent：父级参照物（父级参照物不等价于父级元素：父级参照物和它的父级元素没有直接的关系）

==父级参照物：同一个平面当中最外层的容器是所有底层盒子的父级参照物。==

默认情况下，一个页面中所有元素的父级参照物是body，而==body的父级参照物是null==。

==当我们给元素设置定位之后，会改变元素的父级参照物==（因为设置定位，会让元素脱离平面（脱离文档流））。

下图给最外层盒子设置定位属性，outer脱离正常文档流，outer成为内部inner、center盒子的父级参照物了。outer本身的父级参照物是body。

下图给inner盒子设置定位属性，inner脱离正常文档流，inner成为center的父级参照物。inner的父级参照物还是body，outer的父级参照物也是body。

offsetLeft 和 offsetTop

offsetLeft 和 offsetTop是当前元素的==外边框==距离父级参照物的==内边框==的偏移量（左偏移、上偏移）

标准的IE8浏览器有特殊性：它的偏移量是从当前元素的外边框距离父级参照物的外边框的偏移量。

window.navigator.userAgent获取当前浏览器的版本信息

下图为一般浏览器计算偏移量的示例：

获取页面中任何一个元素距离body的左偏移和上偏移（它的父级参照物是谁不一定）

/*
  思路：
  1、首先获取自己的偏移量，以及自己的父级参照物
  2、如果自己的父级参照物不是body，则我们加上父级参照物的边框和父级参照物的偏移量，一直加到某一次找到的父级参照物是body为止。
  3、标准IE8浏览器不需要加边框（偏移量已经包含了父级参照物的边框了）
  IE8下输出navigator.userAgent可以得到下面的内容
  Mozilla/4.0 (compatible; MSIE 8.0; Windows NT 6.1; WOW64; Trident/7.0; SLCC2; .NET CLR 2.0.50727; .NET CLR 3.5.30729; .NET CLR 3.0.30729; Media Center PC 6.0; .NET4.0C; .NET4.0E)

  只要判断里面是否有'MSIE 8'就能判断时否为IE8浏览器
*/

// 获取当前元素距离body的偏移量，包括左偏移和上偏移
function offset(curEle){
    var l=curEle.offsetLeft,
        t=curEle.offsetTop,
        p=curEle.offsetParent;
    while(p.tagname!=='BODDY'){
        if(!/MSIE 8/i.test(navigator.userAgent)){
            // 如果不是IE8浏览器，则要加边框
             l+=p.clientLeft;
            t+=p.clientTop;   
        }
        l+=p.offsetLeft;
        t+=p.offsetTop;
        p=p.offsetParent;
    }
    return {top: t, left: l};
}

scrollLeft 和 scrollTop

scrollLeft：横向滚动条卷去的宽度

scrollTop：竖向滚动条卷去的高度

存在最大值和最小值：

最小值：0

最大值：`scrollHeight -  clientHeight` 页面真实高度减去一屏幕高度

前面讲的JS盒子模型属性都是”只读属性”：只能通过属性获取，不可以修改属性的值（修改也不生效）

但是这两个属性是==”可读写属性”==：不仅仅可以获取值，还可以修改它的值。

// 所以，我们可以用来设置回到顶部：
document.documentElement.scrollTop=0;
// 为了兼容，我们还要写一次
document.body.scrollTop=0;

这样写太麻烦，我们可以封装一个方法：

// 操作有关于浏览器JS盒子模型，兼容处理
function winBox(attr,value){
    // 设置值
    if(typeof value !== 'undefined'){
        document.document[attr]=value;
        document.body[attr]=value;
        return;
    }
    // 获取值
    return document.documentElement[attr]||document.body[attr];
}

compatibility：兼容性

补充

在获取页面滚动距离或者高度之类的属性的时候，兼容写法是要将document.documentElement写在前面document.body写在后面。

在文档使用了DTD时，document.body.scrollTop的值为0，此时需要使用document.documentElement.scrollTop来获取滚动条滚过的长度；在未使用DTD定义文档时，使用document.body.scrollTop获取值。

也就是说，当我们使用W3C标准（!DOCTYPE HTML）渲染页面的时候，是要通过document.documentElement.xx来取值的（或者设置值）。但是呢，此时我们用document.body.xx也能获取到值（但是这个值是不准确的），所以要先把标准的写在前面，不标准的写在后面。
