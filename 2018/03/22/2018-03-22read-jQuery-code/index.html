2、JQ的核心原理解读

分析源代码。

JQ是一个常用类库（常用那个的DOM库），提供了很多真实项目开发中需要使用的属性和方法（这些方法JQ已经帮我们完善了浏览器兼容处理以及一些细节优化）

jQuery本身是一个类

JQ是基于构造函数模式构建的类库。

var jQuery=function(selector,context){

}

当我们在JS中执行：$()、jQuery(),都是在创建JQ类的实例（\$===jQuery）。这些实例都是一个类数组（我们把这个类数组称之为JQ对象），JQ的实例可以使用JQ原型上提供的共有的属性和方法。

项目中我们把$()称之为JQ的选择器，因为执行这个方法可以传递一个selector参数进去，通过selector我们可以获取到需要操作的DOM元素集合（JQ类数组集合）。传递的第二个1参数context它是当前获取元素的上下文（不传递默认是document，如果传递，我们传递一个JS元素对象即可）

但是把它叫做JQ选择器有点笼统，因为传递的selector支持三种格式：

  • 传递的是一个字符串，就是我们所谓的选择器，能够通过选择器获取到元素
  • 传递的是一个元素对象，它的意思是把JS原生对象转换为JQ对象
  • 传递的是个函数，它代表等DOM加载完成再执行对应的JS代码（类似于window。onload）
  • ……

\$(document).ready(fun)和window.onload的区别

  1. $(function(){})可以在同一个页面中使用多次，多次都生效（所以在使用JQ完成代码的时候，我们一般都会把代码放在回调函数中：首先不仅是等到结构加载完再执行，而且还形成了一个闭包）

    原理：利用了DOM二级事件绑定（可以执行多次），监听的是==DOMContentLoaded==事件（DOM结构加载完成就会触发执行）

  2. window.onload本身就是资源都加载完成才会执行，使用的是DOM零级事件绑定，在同一个页面中只能使用一次。

$.noConflict():如果当前项目中引入了两个类库，都是使用$操作的，为了防止$使用权的冲突，JQ做了一个处理，可以让我们转让$的使用权。

var j=$.noConflict(true); 深度转让，把jQuery和$的使用权都转让了，此时只能使用j()执行了。

filter：同级过滤

JQ中的each

JQ中的each有两种

  1. 写在原型上的each：遍历JQ对象中的每一项
  2. 写在对象上的each：工具方法，可以用来遍历数组、类数组、对象等
  3. 内置的each，其实也是调用原型上的each处理的，只不过JQ在处理的时候会内部自己调用
// 内置each
$('a').addClass('select'); // 我们获取的a可能有很多个，执行一次addClass，相当于给每个获取的a都增加了一个叫做select的样式类（JQ中大部分方法在执行的时候，都会把获取的JQ集合中的每一项调用each进行遍历，把需要操作的任务对每一个遍历的元素进行操作）

$('a').css('width'); // 获取的时候只返回第一个元素的样式（设置走内置each批量处理，获取只处理第一个）

/*-----------------------------------*/

// 原型上的each

$('a').each(function(index,item){
    // 传递参数的顺序和数组内置的forEach顺序相反：ary.forEach(funcion(item,index){...});
    // 获取的a有多少个，回调函数被触发执行多少次：index当前遍历这一项的索引，item是当前遍历这一项的内容

    // $(this) 也是当前遍历这一项，但是属于JQ对象

    $.fn.extend({
         pluginTab:pluginTab;   
    })
}(jQuery);

这里传入了jQuery，在自执行函数里面使用\$，这样可以防止其他类库也使用\$出现冲突。
