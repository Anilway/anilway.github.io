在借用数组原型上的方法，把类数组转换为数组的时候：

在IE低版本浏览器中（IE6~8）arguments可以转换，但是==元素集合或者节点集合这些类数组是无法借用slice转换的==，会报错：Array.prototype.slice：’this’不是JavaScript对象

在JS中，当前行代码报错，后面的代码将不再执行：

console.log(a); // Uncaught ReferenceError: a is not defined
var b =13;
console.log(b);

1.1 JS中的异常信息捕获 try catch finally

  1. 捕获到执行的报错（异常）信息
  2. 防止浏览器抛出异常错误导致下面都不再执行的问题
try{
    // 需要执行的JS代码
}catch(e){
    // 如果try中的JS代码执行发生了异常，就会执行catch中的代码
        // 1、必须要有一个形参变量（我们一般命名为e[error]）
        // 2、e中存储了当前代码执行的异常信息
}finally{
    // 不管try中代码是否出现异常，最后都会执行finally中的操作
        // 1、真实项目中很少使用finally
        // 2、try catch必须成对使用，finally可以不加
}

试试看：

try{

} // 单独使用try，报错：Uncaught SyntaxError: Missing catch or finally after try

/*----------------------------------------*/
try{
    console.log(a); // 报错：Uncaught ReferenceError: a is not defined
}finally{
    var b=13;
    console.log(b);
}
// 虽然加了finally之后，不再出现语法错误，但是并没有达到异常信息捕获的目的，后面的代码依然无法执行。

/*----------------------------------------*/

try{
   console.log(a); 
}catch(e){
   var b=13;
   console.log(b);  // 13
}

// 可以正常执行。

// catch里面必须要有参数，这个参数里面储存了错误信息

try{
    console.log(a);
}catch(e){
    console.log(e.message); // a is not defined 异常信息
}

1.2 手动抛出异常信息

有时候，我们需要捕获到异常信息（try catch），但是上面的代码如果报错，我们希望下面的代码不执行（此时我们需要手动抛出异常信息，来阻止下面的代码执行）

throw new Error('需要抛出的异常信息'); // JS中手动抛出异常信息的方法。new Error()是创建Error这个内置类的一个实例。

常见错误：（Error是错误类，下面的其他各种错误是Error的子类）

错误提示 解释
Error 错误
SyntaxError 语法错误
ReferenceError 引用错误（一般都是未定义或者不存在）
TypeError 类型错误（有两种，一种是数据类型使用错误，比如把数字当做函数用；还有一种是没有这个属性）
RangeError 范围错误（一般都是形成一个无法跳出来的死东西，比如死递归，死循环）

Error是错误类，下面的其他各种错误是Error的子类。所以在手动抛错的时候可以：

throw new Error();
throw new ReferenceError();

扩展阅读：由throw new Error() 引发的探讨

try{
    // 执行代码
    throw new Error('抛错信息'); // 本来上面的执行代码没有出错，在此手动抛错，就能执行下面的语句
}catch(e){
    // 执行代码
}

在一些项目中，可能会用到异常抛错来统计用户使用出错的情况。对于一些严重的错误，会以短信的形式发送给开发团队，方便开发人员及时排除错误。一般短信会发给团队的所有人，当你收到信息的时候，你的boss也会收到信息。当你解决完bug，重新上线，大家也都会收到bug解决的信息。boss可以通过你解决问题的时间和响应时间来对你做出评估。

try{
    console.log(a);
    var b=13;
    console.log(b);
}catch(e){
    console.dir(e); // 把错误信息发送给服务器进行统计
}
