1、 call、apply、bind的区别

call、apply、bind都是天生自带的方法（Function.prototype），所有的函数都可以调取这三个方法。

三个方法都是改变this的指向

1.1 call

// 语法
fn.call(context,para1,...)

// 把fn方法执行，并且让fn方法中的this变为context，而para1...都是给fn传递实参

非严格模式下：

function(num1,num2){
    console.log(this);
}

var obj={fn:fn};
fn();     // this:window
obj.fn(); // this.obj

var opp={};
opp.fn();         // 报错：opp中没有fn这个属性
fn.call(opp);     // this:opp  num1&&num2都是undefiend
fn.call(1,2);     // this：1  num1=2  num2=undefiend
fn.call(opp,1,2); // this:opp num1=1  num2=2

// call方法的几个特殊性
fn.call();          // this:window      num1&&num2都是undefined
fn.call(null);      // this:window
fn.call(undefined); // this:window

严格模式下：

"use strict";
fn.call();          // this:undefined
fn.call(undefined): // this:undefined
fn.call(null);      // this:null

个人理解：

我是这样理解call的。

var opp={};
function fn(){}

// 我们如果直接使用opp.fn()返回undefined
opp.fn(); // undefined

// 如果非要在opp使用fn，就可以使用call
fn.call(opp); // 这样其实就相当于opp.fn();虽然opp本身没有fn方法，但是可以通过call改变fn的this指向，opp就称为了fn的执行主体。

1.2 apply

apply的语法和call基本一致，作用原理也基本一致。唯一的区别：apply把传递给函数的实参以数组的形式存放（但是也相当于在给函数一个个的传递实参值）。

补充：apply也可以传递arguments

fn.call(null,10,20,30);
fn.apply(null,[10,20,30]); //  传递给fn的时候也是一个个的传递进去的

1.3 bind

也是改变this的方法，它在IE6,7,8下不兼容；它和call以及apply改变this的原理不一样。

fn.bind(opp)：fn调取Function.prototype上的bind的方法，执行这个方法==返回了一个匿名函数==

fn.call(opp,10,20); // 把fn执行，让fn中的this变为opp，并且把10和20分别传递给fn

fn.bind(opp,10,20); // 预先让fn中的this指向opp，并且把10和20预先传递给fn，此时的fn没有被执行（只有当执行的时候this和实参才会起到应有的作用）

// 需求：点击box这个盒子的时候，执行fn，并且让fn中的this指向opp
oBox.onclick=fn; // 点击的时候执行了fn，但是此时fn中的this是oBox

oBox.onclick=fn.call(opp); // 绑定事件的时候就已经把fn立即执行了（call本身就是立即执行函数），然后把fn执行的返回值绑定给事件

oBox.onclick=fn.bind(opp);

/*
  fn.bind(opp)：fn调取Function.prototype上的bind的方法，执行这个方法返回了一个匿名函数（简化来写，如下）

  function(){
      fn.call(opp);
  }

  oBox.onclick=function(){
      // this:oBox
      fn.call(opp);
  }
*/

bind涉及到柯理化函数思想。

个人总结：

fn.call(obj,xx,xx);
fn.apply(obj,[xx,xx]);

call和apply改变函数this的同时，会立即执行函数，我常直接看成obj.fn()。它们的区别在于传递参数的方式不一样，call是一个个传递参数；apply是把参数写在数组里面，也可以通过arguments传递。

var fun=fn.bind(obj,xxx,xxx);
fun();

通过bind绑定函数，改变函数的this，不会立即执行函数，而是返回一个匿名函数，类似于：

function(){
    fn.call(obj,xxx,xxx);
}

然后你再去执行。bind方法可以用在事件绑定上，我理解为类似于事件委托的作用：oBox.onclick=fn.bind(oLi)。

bind方法不兼容IE6~8。

1.4 练习

接下来我们通过几道练习题加深理解。

function fn1(){
    console.log(1);
}
function fn2(){
    console.log(2);
}
fn1.call(fn2);
fn1.call.call.call(fn2);
Function.prototype.call(fn2);
Function.prototype.call.call.call(fn2);

题目分析：

首先我们看看call方法的实现方式：

Function.prototype.call=function call(context){
    // call函数本身的this指向调用它的那个函数。
    // 比如：fn1.call(fn2)中call本身的this是fn1

    // 注意下面的描述，要分清楚是call的this（fn1），和call的this的this（fn1的this）。

    // [native code]做了两件事：
    // 1、把call的this中的this指向context（按上面的例子来说，就是把fn1中的this指向fn2）
    // 2、执行call中的this：this();（按上面的例子，就是fn1();）
}

fn1.call(fn2);输出结果分析：

/*
  fn1调用call方法，call方法中传递了实参fn2。
  call方法的this是fn1.（这个和平时判断this是一样的）

  call方法执行做了下面两件事：
  1、让call的this的this指向fn2 ——> 让fn1的this指向fn2
  2、执行call的this ——> 执行fn1()

  因为fn1中的代码只是单纯的输出，没有涉及到this的相关操作，所以，还是输出1.

    fn1.call：fn1这个Function的实例通过__proto__找到Function.prototype上的call方法，然后让call方法执行（传递fn2这个实参）

  执行call的时候，call中的this：fn1，所以此处是把fn1执行，让fn1中的this指向fn2（只不过我们的fn1中不需要使用this）


  这个很简单，其实一看就知道。但是我们通过去研究call的实现方式去更详细的了解很有必要。在后面会对call进行更多的改变，方便我们到时候去理解。


*/

fn1.call.call.call(fn2);的输出结果分析：

/*
  这个分析起来会稍微抽象一点。但是也不复杂，就是容易混淆。我们先来单纯的看一看这个代码：fn1.call.call.call(fn2);
  fn1.call:
      就是fn1访问了Function.prototype中的call方法。

  fn1.call.call:
     相当于(fn1.call).call，就是call方法再访问，或者说再调用call方法。因为call本身就是函数，所以，可以通过原型链找到call方法，其实还是它本身。

   fn1.call.call.call：
      这个分析同理。
*/


/*
  从上图可以看出，call本身调用call，通过原型链查找，还是找到的它自己。就是类似于call.call这种的。把这个弄清楚了，就容易分析了。

fn1.call.call.call(fn2)这个代码首先就是[fn1.call.call]函数调用call方法，传递实参fn2。

[fn1.call.call]函数实际就是call函数，fn1通过原型链找到call，call再通过原型链找到call。那也就是call方法调用call

那整体就是call方法调用call，传递了实参fn2：

    1、把call的this（也就是fn1.call.call）的this指定为fn2。因为前面分析了[fn1.call.call]函数实际就是call函数，所以，就是把call函数里面的this指向fn2；
    2、执行call的this。当前这次call函数的this还是call（也就是fn1.call.call）。（因为当前call函数执行的时候，从堆内存里面提取的函数代码中的this没有改变，所以当前函数栈内存中的执行代码中，this还是call。虽然当前call执行的时候，在上面一步把call的this变为fn2，但是那是在当前call执行之后才改变堆内存中的this的，并不会改变当前call栈内存中的执行代码中的call）

    3、执行fn1.call.call();
        3.1 call的this此时却是为fn2了，因为没有传参，所以fn2的this还是fn2；
        3.2 执行call的this，也就是执行fn2，输出2.
*/

// 我的疑问：call中的this被改变了，那是不是说，把原型里面call函数的代码改变了？如果是这样的话，就会所有的函数调用call方法都会受到影响了。

个人感悟：

==我们去看call的实现方式是：首先将call的this的this指向传递的实参，然后再执行call的this。==

==一般情况下，我们可以认为fn1.call(fn2)相当于给fn2添加了fn1方法，在执行fn2.fn1()。因为call的this并没有改变，最后执行的还是call原本的this。==

但是在上面的题目中就出现call的原本this发生改变的情况：

fn1.call.call(fn2);

这里call的原本this是fn1.call，但是当call执行的时候，会首先把call的this指向其传递的实参，在这里就是把fn1.call的this指向fn2。这里就会出现一个有意思的情况：

虽然本次call的this没有改变，还是fn1.call；但是继续执行fn1.call的时候，此时的call的this就是fn2了，又因为没有传参数进去，所以fn2的this还是fn2，最后再执行的是fn2.

Function.prototype.call(fn2);输出结果分析：

我们先要了解一个小知识点：

  • ==Function.prototype是function类型的==

我们通常都认为prototype是个对象类型，是’object’类型的，这到没错。但是对于Function类来说，它的原型也是function类，因为我们使用的匿名函数就是通过这个原型实现的，这个原型本身也是匿名函数。

也就是说，Function.prototype是个函数，是可以被执行的。

我们在原型使用的时候，还是和使用一般的原型一样。

/*
  Function.prototype.call(fn2)就是Function.prototype调用call函数：
      1、call函数执行，把call的this（Function.prototype）中的this指向fn2；
      2、执行call的this，也就是Function.prototype()。因为函数类的原型是一个匿名空函数，没有输出。所以最后的结果为无输出。


这里的输出不要和返回值弄混淆了，原型的返回值肯定是有的，为undefined，但是你要主动获取，不会直接输出给你。

这里的没有输出是说没有主动输出的代码，比如console.log之类的，所以执行结果就是没有输出。
*/

Function.prototype.call.call.call(fn2);输出结果分析：

其实就和前面fn1.call.call.call(fn2);是同样的道理。call的this被改变，指向fn2，call执行完后，再执行call的this，就是执行fn2，输出2。

call练习2：

var test = function(){
  console.log('hello world')
  return 'fsjohnhuang'
}
test.call() // ①
Function.prototype.call(test) // ②
Function.prototype.call.call(test) // ③
Function.prototype.call.call(Function.prototype.call, test) // ④

答案：①、③和④. 控制台显示hello world，并返回fsjohnhuang。②. 返回undefined且不会调用test函数

2、获取数组中的最大值和最小值

方案一：先排序，后取最大值和最小值

var ary=[12,45,23,34,16,28,11];
ary.sort(function(a,b){
    return a-b;
});

var max=ary[ary.length-1],
    min=ary[0];
console.log(max,min); // 45 11

方案二：假设法

先假设第一个是最大值，然后循环数组中的每一项，和假设值进行比较，如果当前遍历的这一项比假设的值大，我们替换假设的值即可。

var ary=[12,45,23,34,16,28,11];
var max=ary[0],
    min=ary[0];

for(var i=1;i<ary.length;i++){
    max=max<ary[i]?ary[i]:max;
    min=min>ary[i]?ary[i]:min;
}

console.log(max,min); // 45 11

方案三：Math.max()和Math.min()

第一次尝试：把数组转换为字符串，然后通过eval转换执行。

var ary=[12,45,23,34,16,28,11];
Math.max(ary.toString); // Math.max("12,45,23,34,16,28,11") 结果为NaN

// Math.max()中的参数一定要是12,45,23,34,16,28,11这种才行，用eval()行不行呢？
eval("12,45,23,34,16,28,11"); // 结果返回 11；这是为什么呢？这是因为括号表达式中如果出现多项，只操作最后一项，见下面的详解

JS中的括号表达式：

  1. 小括号当中出现多项（每一项之间用逗号分隔），操作的只有==最后一项==。

    (function(){
    console.log(1);
},function(){
    console.log(2);
})() // 2 只把第二个函数执行了，第一个不操作

console.log((12,13,14,23,34,56,78,12,66)); // 66
eval('(12,13,14,23,34,56,78,12,66)');      // 66
eval('12,13,14,23,34,56,78,12,66');        // 66

  2. 括号表达式会==影响this的指向==

    • 括号当中只有一项，this还是按照没加括号的时候处理即可。
    • 括号当中有多项，执行最后一项的时候，方法中的this就是window（严格模式下this是undefined）[浏览器把这种情况理解为自执行函数执行处理的]
    window.name='win';
function fn(){
    console.log(this.name);
}

var obj={name:'Joy',fn:fn};
obj.fn();      // 'Joy'
(obj.fn)();    // 'Joy'
(fn,obj.fn)(); // 'win'  在严格模式下fn的this是undefiend，会报错

小括号在JS中的作用还有很多：

  • 函数的执行会用到小括号
  • 定义函数，传递形参的位置也是小括号
  • 改变运算的优先级
  • …

第二次尝试：先把整个代码用字符串拼接，在通过eval执行

/*
   先使用字符串拼接，把需要执行的语句拼接成为字符串，最后eval一下即可。

   真实项目中尽量少使用eval（防止代码压缩成为一行后，eval导致代码结构或者执行混乱）。
*/

var ary=[12,45,23,34,16,28,11];
var max=eval('Math.max('+ary.toString()+')');
var min=eval('Math.min('+ary.toString()+')');

console.log(max,min); // 45 11

方案四：使用apply

var ary=[12,45,23,34,16,28,11];

// apply的特点：虽然编写的是一个数组，但是也相当于在执行函数的时候一项一项的传递参数
// 这里用不到this，第一个参数可以随便定义。要保持一致就写Math
var max=Math.max.apply(Math,ary);
var mix=Math.min.apply(Math,ary);

console.log(max,min); // 45 11

方案五：ES6语法实现

利用ES6的==展开运算符==来实现需求（原理：和apply这个一样）

let ary=[12,45,23,34,16,28,11];

let max=Math.max(...ary),
    min=Math.min(...ary);
console.log(min,max); // 11 45

3、获取平均数

最基本的实现方式：

// 去除最高和最低分数，剩下的值求平均数
function avgFn(){

    // 1、先把实参集合进行排序，去除第一个和最后一个
        // arguments是类数组集合，不能使用数组的sort方法

            // 先把arguments转换为数组：
        var ary=[];
        for(var i=0;i<arguments.length;i++){
            ary[i]=arguments[i];
        }
        // 对数组排序，去除最大值和最小值
        ary.sort(function(a,b){return a-b;});
        ary.shift();
        ary.pop(); // 或者ary.length--;

    // 2、把集合中剩下的值求平均数，求和后初一长度即可
        var sum=0;
        for(var j=0;j<ary.length;j++){
            sum+=ary[j];
        }
        return (sum/ary.length).toFixed(2); // 保留两位小数
}

avgFn(9.8,9.3,9,8.8,9.7,9.6); // 9.40

基本优化：

function avgFn(){
    var ary=[];
    for(var i=0;i<arguments.length;i++){
        ary[i]=arguments[i];
    }
    // 链式写法
    ary.sort(function(a,b){return a-b;}).shift();
    ary.length--;

    // 利用join和eval求和
    return (eval(ary.join('+'))/ary.length).toFixed(2);
}

利用继承优化：（不兼容IE）

function avgFn(){
    arguments.__proto__=Array.prototype; // IE浏览器下不兼容（IE下为了保护原型链进制我们使用__proto__）
    arguments.sort(function(a,b){return a-b;}).shift();
    arguments.length--; 
    return (eval(arguments.join('+'))/arguments.length).toFixed(2);
}
avgFn(9.8,9.3,9,8.8,9.7,9.6); // 9.40

利用继承优化：（兼容IE）

/*
借用数组原型上的slice方法，实现将类数组转换为数组
Array.prototype.slice、[].__proto__.slice、[].slice都可以获得slice方法

原理：
执行数组原型上的slice方法，让方法中的this指向要转换的类数组，这样在执行内置代码的时候，this已经变为arguments，相当于操作arguments。

前提：
必须是类数组才可以，因为类数组虽然不是数组，但是它的结构和数组基本上类似，也就是操作数组的那些循环判断等JS语句，同样也能操作arguments这种类数组。

类数组不仅可以借用slice，Array原型上的大部分方法都可以借来使用（原理都是this改变）

 */
function avgFn(){
    var ary=Array.prototype.slice.call(arguments);
    ary.sort(function(a,b){return a-b}).shift();
    ary.length--;
    return (eval(ary.join('+'))/ary.length).toFixed(2);
}
avgFn(9.8,9.3,9,8.8,9.7,9.6); // 9.40

复习一下数组的克隆：

// 数组克隆：
ary.slice(); // 即可实现数组克隆。

// 如果让你在实现slice克隆的方法，你会怎么写呢？
Array.prototype.slice=function slice(){
    var ary=[];
    for(var i=0;i<this.length;i++){
        ary[ary.length]=this[i];
    }
    return ary;
}

不把arguments转换为数组，借用数组中的方法实现：

function avgFn(){
    [].sort.call(arguments,function(a,b){
        return a-b;
    });
    [].shift.call(arguments);
    [].pop.call(arguments);
    return (eval([].join.call(arguments,'+'))/arguments.length).toFixed(2);
}

avgFn(9.8,9.3,9,8.8,9.7,9.6); // 9.40
